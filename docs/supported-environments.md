# 対応環境マトリクス（NF-038）

**最終更新:** 2026-07-21
**目的:** Pitchsieve（採譜エンジン）の動作対象環境と検証状況を明示する。

## OS × Python

| OS | Python 3.12 | Python 3.13 | Python 3.14 | 状況 |
|---|---|---|---|---|
| macOS (Apple Silicon) | ✅ | ✅ | ✅（推奨・CI相当） | 主要開発環境で検証 |
| macOS (Intel) | ✅ | ✅ | ✅ | 同一コードパス（未実機常設） |
| Linux (x86_64) | ✅ | ✅ | ✅ | 依存は全て対応。CI相当で緑 |
| Windows | ⚠️ 未検証 | ⚠️ 未検証 | ⚠️ 未検証 | パス処理は `pathlib` 使用だが未実機検証 |

- **メインエンジン**: Python **3.12 以上**（3.14 推奨）。
- 音声 I/O は `soundfile`（libsndfile 同梱）で OS 差を吸収。

## 機能別の追加要件

| 機能 | 追加要件 | 対応 Python | 備考 |
|---|---|---|---|
| 単旋律採譜（既定・`mono`） | コア依存のみ | 3.12+ | pYIN |
| 多声採譜（`--engine poly`） | basic-pitch（別 venv） | **3.12 専用** | Python 3.13+ 非対応のため subprocess 分離 |
| ステム分離（`--stem`） | demucs（別 venv） | 3.12 | 要 PyTorch |
| GP5 出力（`--format gp5`） | PyGuitarPro | 3.12+ | LGPL |
| マイク録音（`record`） | sounddevice + PortAudio | 3.12+ | ハードウェア必須。CI 非対象 |
| 浄書 PDF（`--pdf`/`--tab`） | verovio wheel（同梱） | 3.12+ | |

## ハードウェア／実行時

| 項目 | 目安 |
|---|---|
| メモリ | 単旋律で 1GB 程度、多声(basic-pitch)は 2–4GB |
| ネットワーク | **不要**（完全ローカル処理・NF-001） |
| GPU | 不要（demucs 使用時のみ任意で高速化） |
| 対応入力形式 | wav / mp3 / flac / ogg / m4a（librosa が読める形式） |
| 対応楽曲長 | 2時間保証・最大24時間（長尺は `chunk` で分割推奨） |

## 検証の限界（正直な記録）

- **Windows は未実機検証**。パス・改行・音声デバイス周りで差異が出る可能性がある。
- 多声・ステム分離は別 venv（Python 3.12）を要し、無い環境では `mono` に正直にフォールバックする。
- `record`（マイク）は物理デバイス依存のため CI では境界（未導入時の失敗）のみ検証。

> 環境ごとの受入は、`NF-043 推論ランタイム別受入ゲート` の整備と合わせて今後拡充する。
