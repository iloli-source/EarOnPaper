# 対応環境マトリクス（NF-038）

**最終更新:** 2026-07-24
**目的:** Pitchsieve（採譜エンジン）の動作対象と、**実際に確認できた範囲**を分けて示す（D4M-030: 「想定対応」と「実測証跡あり」を混同しない）。

## OS × Python（証跡ベース）

| OS | Python 3.12 | Python 3.13 | Python 3.14 | 現在の証跡 |
|---|---|---|---|---|
| macOS (Apple Silicon) | ✅ 実測 | ⚠️ 未実測 | ✅ 実測（主開発機） | 3.14: フルpytest(1242)・Playwright E2E・実音源E2Eを日常実行。3.12: basic-pitch/demucs別venvのworker経路として日常使用 |
| macOS (Intel) | ⚠️ 未実機 | ⚠️ 未実機 | ⚠️ 未実機 | 同一コードパスだが実機証跡なし |
| Linux (x86_64) | ⚠️ 未再現 | 🟡 部分確認 | ⚠️ 未再現 | 外部検証(2026-07-23)でPython 3.13.5・依存不要範囲727件通過。music21/verovio含む正式依存環境は未再現 |
| Windows | ⚠️ 未実機 | ⚠️ 未実機 | ⚠️ 未実機 | パス処理・プロセス停止・Electron動作を実機確認していない |

- 「対応表示」は、対象環境でクリーン導入→全pytest→実音源E2E→成果物確認を通過してから行う。
- `requirements.txt` のバージョン固定は導入・動作確認済みの証明ではない。

## 検証の限界（正直な記録）

- **Windows・macOS Intel・macOS上のPy3.13は未実機/未実測**。
- Linuxの証跡は外部検証の部分実行のみ（正式依存なし）。
- 多声・ステム分離は別 venv（Python 3.12）を要し、無い環境では `mono` に正直にフォールバックする。
- `record`（マイク）は物理デバイス依存のため CI では境界（未導入時の失敗）のみ検証。

> 環境ごとの受入は、`NF-043 推論ランタイム別受入ゲート` の整備と合わせて今後拡充する。
