# AI採譜デスクトップアプリ UI/UX 分解調査（Fable）

**調査日:** 2026-07-19
**目的:** 「直感的で、色で操作が区分され、マイクロインタラクションがあり、操作するのが気持ちよい」UIを目指すための競合・優良事例の分解と、採譜特有UX課題への設計案。
**接続する既存要件:** 編集UIカテゴリ（F-038 下書きUX / F-042 原音源同期再生 / F-043 信頼度ハイライト=色+形状の二重符号化 / F-044 仕上げチェックリスト / F-080 五線+TAB連動）、NF-033（アクセシビリティ）、NF-041（UI応答時間）。本調査はこれらと整合させ、色区分・気持ちよさ・待ち時間体験を具体化することを狙いとする。

**分解から得た横断的教訓:**
- 「モダンな見た目」だけでは不十分。MuseScore 4の失敗は**情報を隠して掘らせたこと**。採譜では信頼度・要確認箇所を"掘らせず"常時可視にすべき（F-043との整合）。
- Doricoのモード分離は誤操作防止に有効だが、採譜アプリでは明示モードを強制せず**「下書き→検証→浄書」の暗黙ガイド**（F-065ワークフローガイド）が望ましい。
- Klangioのデュアルビュー（五線+TAB）は本アプリのF-080と方向一致。差別化は**待ち時間体験・信頼度可視化・気持ちよさ**で作る。

## 2. 受賞デザインからの学び

**Moises（Apple Design Award 2025 ファイナリスト）の4原則: Human / Universal / Simple / Efficient。**
- **Simple = 摩擦除去**: 機能追加でなく「直感的でシームレスな操作」を優先。ステム分離という本来スタジオ機能を最小ジェスチャに。→ 採譜も「D&D→即プレビュー結果」を軸に、設定は隠して必要時に開く。
- **Efficient = 創作フローを断たない**: 再生開始でコードカードが画面を占有し楽曲と完全同期。メニュー移動を排除。→ 本アプリのF-042原音同期・F-087プレイアロングに直結。
- **Universal = 初日から33言語**: 多言語を後付けでなく設計前提に。→ F-068日本語UI/F-069多言語との整合。
- **技術の隠蔽**: 高度技術を「クリーンでアクセシブルな体験」に変換し、ユーザーは学習でなく創作に集中。→ **解析エンジンの複雑さをUIに漏らさない**のが受賞水準。

**Linear（"触って気持ちいい"SaaSの規範）:**
- **速度は機能**: 楽観的UI・スケルトン・積極キャッシュで遷移<100ms。「工学速度=単一操作の速さ / 設計速度=各操作までの距離の短さ」。→ NF-041のUI応答時間目標と整合。編集操作は楽観的反映（先に見た目を変え、確定を裏で処理）。
- **キーボード第一**: 全主要操作にショートカット、コマンドパレット1打鍵。→ 高速編集（F-047）の指針。
- **4pxグリッド+抑制配色**: 全要素が4pxグリッドに整列し「秩序」を脳が処理。テキスト/アイコンは40-60%不透明度に沈め、**状態・優先度・操作可能要素のみフル彩度**。→ 採譜の配色設計に直輸入すべき原則（下記4章）。

## 3. 「気持ちよさ」の構成要素（juice の分解）

ゲームUIの"juice"とマイクロインタラクション論から、採譜アプリに移植可能な要素:

1. **即時フィードバック**: 全操作に視覚（チェック/色遷移）+聴覚（心地よいチャイム/控えめなエラー音）+触覚（haptic tap）。Duolingoは正誤に緑チェック/赤バツ+祝祭アニメ。→ **音符確定・要確認クリア・書き出し完了**に祝祭的マイクロ演出。
2. **視覚的な内臓感覚（visceral）**: バウンスアニメ・色遷移そのものが「意味を考える前に気持ちいい」。squash&stretchで有機的に。
3. **粒子・軽量エフェクト**: 塵・きらめきで平坦な操作に運動感。採譜では**音符を"確定済み"にした瞬間の淡い光跡**など控えめに。
4. **サウンドデザイン**: 物理性と情感を付与。ただし採譜アプリは原音再生が主役なので**UI効果音は極小音量・非侵襲**。
5. **手応え（weight）**: スクリーンシェイクの音楽版=強い操作に微小な視覚的重み。誤操作アンドゥ時などに。
6. **応答するUIの磨き**: Linear的な「常に前進するプログレス」「stall しないバー」。停滞するバーはユーザーが最も嫌う。

**設計上の節度（重要）:** juiceは"追加"ではなく"感じ方の変化"。過剰な演出・感覚過負荷は逆効果。採譜は長時間の集中編集作業のため、**祝祭は達成の瞬間に限定し、常用操作は静かで速い**を原則とする。

## 4. 採譜特有UXの設計案

### 4-1. 解析待ち時間（数分）の体験設計
知覚待ち時間は実時間より重要。10秒超はプログレスバー+パーセント+状態文が推奨。
- **常に前進するバー**: 停滞禁止。段階ラベルを状態文で更新（例:「ステム分離中→拍・小節推定中→整譜中」＝F-003/F-012/F-021の内部工程を可視化）。
- **部分結果の先出し（progressive rendering）**: 全曲完了を待たせず、確定した先頭セクションから五線/TABを描画開始。「システムが動いている」実感と早期の操作開始を両立（skeleton→本描画）。採譜は多段パイプラインなので**セクション単位の逐次表示**が自然（F-004長尺分割と接続）。
- **待ち時間の教育コンテンツ化**: F-085「手直しが多い可能性」の事前提示、F-065ワークフローガイドの4ステップ、耳コピ学習（F-064）への誘導を待機画面に。ただし押し付けず、スキップ可能に。
- **速度/精度スライダー**（F-067）: 待ち時間そのものをユーザー制御下に置き「待たされ感」を主体感に変える。

### 4-2. AI結果の信頼度可視化（F-043の具体化）
- **二重符号化必須**（NF-033）: 色（緑=高/琥珀=中/赤=低）+**形状/テクスチャ**（実線=確定、破線/ハッチ=要確認）。色覚多様性に対応。
- **常時可視・掘らせない**（MuseScore 4の反面教師）: 要確認箇所は編集画面に常に見え、F-042の「不一致箇所へジャンプ」導線と連動。
- **過負荷回避**: 生の0.73を出さず「要確認」ラベル/バーで表現。研究上、不信ユーザーの58%が「不確実性の明示」で信頼を回復。
- **下書きバッジ**（F-038）: AI生成明示・未確認/確認済みの視覚区別・出力前の未確認警告。

### 4-3. ビュー切替（五線/TAB/ピアノロール/簡譜）
- Klangio型デュアル（五線+TAB連動=F-080）を基本に、**ピアノロール/スペクトログラム（F-040/F-041）はタブ or スライドインで併置**。
- Doricoのモード分離は採用しないが、**「聴いて確認」「音符を直す」「浄書して出す」の3局面**を色/アイコンで緩く区別（強制モードにはしない）。

### 4-4. 聴きながら確認する再生同期ハイライト（F-042）
- 再生ヘッド追従カーソル+現在音符ハイライト+自動スクロール（sheet music appの定番UX）。
- Moisesの「コード進行が画面を占有し楽曲と同期」を採譜検証に応用: 検証モードでは**要確認音符へ自動ジャンプ+その箇所をループ**（F-061区間ループ+F-059減速再生とバンドル）。
- テンポ変更（1/20〜2倍速）でAIドラフトの耳検証環境を構成。

## 5. 採譜アプリが目指すべきUX方向性の仮説3案

### 案A: 「静かな工房」— Linear × Dorico系
落ち着いた抑制配色（テキスト/アイコンは低不透明度に沈め、**信頼度・状態・操作可能要素だけがフル彩度で発光**）。4pxグリッドの秩序、キーボード第一の高速編集、遷移<100ms。祝祭演出は達成の瞬間のみ。
**雰囲気の言語化:** 「深夜のスタジオ。余計な光を消し、直すべき音符だけが静かに灯る。速く、正確で、邪魔をしない。プロの長時間編集に耐える集中の道具。」
**強み:** NF-041/NF-033と最も整合。長時間編集の疲労が少ない。**リスク:** 「気持ちよさ」が地味に振れると要望から外れる。

### 案B: 「弾ける歓び」— Moises × Duolingo系
明快でカラフル、色で操作を強く区分（青=操作/緑=確定/琥珀=要確認）。マイクロインタラクションと祝祭アニメを前面に。D&D→即プレビュー、プレイアロング（F-087）で"弾けた"体験を主役に。待ち時間も演出で楽しませる。
**雰囲気の言語化:** 「音源を放り込むと画面が呼吸し、音符が light up する。直すたびに小さな祝福があり、弾けるようになる過程そのものが楽しい。触っていたくなる採譜。」
**強み:** ユーザー要望「操作が気持ちよい」に最も直球。差別化しやすい。**リスク:** 長時間編集で演出が過剰・うるさくなる恐れ→祝祭の頻度設計が肝。

### 案C: 「進化する下書き」— 信頼度と部分結果を主役に
本アプリ固有の「下書きUX（F-038）+信頼度（F-043）+部分結果先出し」を体験の中心に据える。解析中からセクションが逐次立ち上がり、要確認箇所が呼吸するようにハイライトされ、直すたびに"確定済み"へと色が育つ。待ち時間・不確実性・編集を一つの連続体として見せる。
**雰囲気の言語化:** 「もやから輪郭が立ち上がる。AIが確信のない箇所を正直に差し出し、あなたが触れるたびに譜面が確かになっていく。"仕上げていく"手応えが可視化された、正直で伸びやかな採譜体験。」
**強み:** 競合が持たない差別化軸（信頼度×部分結果）と直結。V5の下書きコンセプトを最大化。**リスク:** 実装難度が高く、部分結果描画・逐次信頼度更新の技術前提を要する。

**推奨:** 案Aの規律（配色・速度・アクセシビリティ）を土台に、案Cの「進化する下書き」を差別化の核とし、案Bの祝祭マイクロインタラクションを"達成の瞬間限定"で味付けするハイブリッドが、ユーザー要望と既存要件（F-038/043/042・NF-033/041）の両立に最も適する。
