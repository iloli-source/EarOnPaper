# Web網羅ギャップスキャン v2 — v2要件でもなお欠落するもの

**調査日:** 2026-07-19
**調査担当:** 要件ギャップ調査（Web実ユーザー情報源・第2巡）
**対象:** functional-requirements.md v2（機能97件 F-001〜F-097）+ non-functional-requirements.md v2（非機能41件 NF-001〜NF-041）
**方針:** v2改訂で対応済みの発見（前回G-01〜06・U-01〜03・C-01〜07）とは重複しない、**v2でもなお欠落する証拠のあるニーズ**のみを新規に列挙する。証拠のない項目は書かない。前回カバレッジの穴（Reddit生スレ・VI-Control・中華圏・note/Qiita・編成別コミュニティ）を優先探索した。

> **前提の再確認:** v2は前回スキャン6発見（異名同音F-081/装飾音F-082/rebarring F-083/MIDIドラフトF-084/採用可否F-085/密度F-095、テンポマップF-017 Must昇格、TAB群F-076〜080）を的確に取り込んでおり、Web上の最頻不満（ルバート・11/8・非現実運指・変則チューニング・英語UI・想定外請求・倍音誤認）はほぼ全て要件化済み。以下は「それでも漏れているもの」に厳しく絞った。**新規欠落は2件のみ**であり、これはv2の網羅性の高さを裏づける結果でもある。

## 1. v2でもなお欠落 — 新規発見

### N-01: 移動ド（movable-do ソルフェージュ）記譜出力
- **根拠（情報源と要旨）:**
  - Qiita「好きな曲で相対音感を鍛える！Demucs + Basic Pitch で移動ド譜を自動生成するツールを作った」— 相対音感トレーニングは労力が大きく初心者の継続が困難という課題に対し、「調が変わっても同じパターンとして認識できる」移動ド譜を自動生成する専用ツールを自作。五線譜・TABは絶対音高を示すが、移動ドは中心音基準の相対関係のみに集中させる点で本質的に異なると明記。
  - note.com「音楽の耳コピむずかしいですよね」他、耳コピ学習文脈で相対音感・移動ドを軸にした記事が複数存在。
  - 個人が既存ツールに存在しないため自作している事実そのものが未充足需要のシグナル。
- **提案要件名:** 移動ド／ソルフェージュ記譜出力。推定キーを基準に音名を「ド基準の相対音度（ドレミ/数字/Do-Re-Mi）」で表示・出力し、調が変わっても同一パターンで読める相対音感トレーニング向け記譜。
- **提案カテゴリ:** 記譜形式（F-031〜037系の隣接、F-033简谱の兄弟要件）
- **提案MoSCoW:** Could / Phase3（耳コピ学習モードF-064と接続する周辺価値。中核ではない）
- **既存IDでカバーできない理由:** F-033（简谱=固定ド数字譜）は絶対的な階名で移動ドとは別概念。F-091（度数/ローマ数字/Nashville）はコード進行の度数表記でありメロディの階名表記ではない。F-064（耳コピ学習モード）は答え開示UIで記譜形式ではない。**移動ド=メロディの相対階名記譜**を持つIDは存在しない。

### N-02: 多声ボーカル／合唱（SATB等）のパート別声部採譜
- **根拠（情報源と要旨）:**
  - transcribeasong.com「Choir Transcription」・mysheetmusictranscriptions.com「Vocal Ensemble Transcription Service」・tunescribers（SATB/TTBB/SAT等の声部指定で note-for-note 採譜を受注）— **合唱・ボーカルアンサンブルの声部別採譜が有償の人的サービスとして独立市場を形成**している事実。
  - partwriter.com・gadegetkit「Multi-part Harmony Desk」— リード旋律からSATB四声を生成する自動パートライター、MuseScore「Check Harmony Rules」プラグイン等、四声処理に特化したツール群が存在。
  - Qiita「アカペラアレンジを少しでも自動化する試み」— アカペラ制作で「採譜（耳コピ）工程でとにかく時間を食われる」と明記、複数声部・和声構造認識の自動化需要を示す。
- **提案要件名:** 多声ボーカル声部採譜。混合ボーカル／合唱音源からソプラノ・アルト・テナー・バス等の声部を分離し、声部指定（SATB/TTBB/SAT等）に基づきパート別譜面を生成。受入条件は2-4声の小編成に限定。
- **提案カテゴリ:** 解析エンジン（F-020歌声採譜の拡張、F-019多声部の声楽版）
- **提案MoSCoW:** Could / Phase3（人的サービス市場が存在＝需要は実在。ただしF-019と同じ最難関領域のため小編成限定・観測扱い）
- **既存IDでカバーできない理由:** F-020（歌声採譜・歌詞同期）は単一ボーカル旋律＋歌詞が主眼で多声声部分離を含まない。F-019（多声部一括採譜）は器楽アンサンブル前提で受入条件も器楽小編成。**ボーカルハーモニーの声部分離＝合唱/アカペラ特化**の採譜を明示するIDはない。器楽と声楽では分離アルゴリズムも記譜慣行（stem方向・2段譜割当）も異なるため、F-019/F-020の受入条件を声楽に拡張する記述追加でも可。

## 2. v2の変更の妥当性検証

### 2-1. Won't降格6件を惜しむ声の有無
第2巡の探索でも、以下の降格を覆す実ユーザー需要の一次証拠は**得られなかった**。降格は妥当と評価する。

- **F-026 難易度別アレンジ / F-095 密度簡略化:** 教育文脈（Songscription for-teachers）では「simplify the parts」が実需としてあるが、これはF-095密度簡略化スライダー（Could維持）で代替されており、基礎精度未達での難易度アレンジ提供を惜しむ声は見つからず。降格の論理（精度不信の増幅回避）は実証寄り。
- **F-027 ピアノリダクション:** VI-Control のオーケストラ/劇伴スレッドを確認したが、話題は「AIステム分離が重なるパッド系（strings/brass/woodwinds）を分離できない」という**前段の分離限界**であり、リダクション自動化への需要ではなかった。前提F-019が研究段階である以上、降格は妥当。
- **F-035 sargam / F-056 DAWプラグイン / F-062 スマートメトロノーム:** これらを惜しむ一次証拠は今回も皆無。降格を覆す材料なし。

### 2-2. TAB系昇格の需要強度
TAB必須化・エンジン群昇格を裏づける証拠が第2巡でさらに積み増しされた。**昇格は強く支持される。**
- La Touche Musicale が「生徒がYouTube曲を土壇場でリクエスト」する教師向けにTAB＋MusicXML出力を訴求。Klangio Transcription Studio も multi-instrument sheet music/MusicXML/MIDI/**TABs** を明示的な柱に。中国 zhihu でも「吉他谱」は主要ツールの評価軸。TABは英語圏・中華圏・日本語圏いずれでも一次需要が確認できる。

### 2-3. 見落としの兆候
- **移動ド（N-01）**が唯一「個人が自作している＝既存ツールに無い」明確な兆候。ただしCould相当で中核ではない。
- **合唱声部（N-02）**は有償人的市場が存在する点で需要の実在は堅いが、技術的難度からPhase3観測が妥当。
- 中華圏の一次挙動として、反谱AI（serocs.cn）が「音楽転譜＝五線譜と**简谱**を並行出力」を既定にしている点は、F-033简谱の需要強度（中華圏では简谱が第一級市民）を裏づける。F-033のShould/Phase3据え置きは妥当だが、中華圏展開を本気で狙うなら简谱は五線と同格の出力という位置づけを実装フェーズで再確認する余地。

## 3. 結論 — v2は「本当に良い」か

**v2は本当に良い。** 第2巡で前回未踏の情報源（VI-Control・note/Qiita日本語実装記事・中華圏 zhihu/serocs・合唱/ワーシップ/教育の編成別コミュニティ・Product Hunt launch）を横断しても、v2の97+41件を覆す構造的欠落は出てこなかった。新規欠落は移動ド記譜（N-01）と多声ボーカル声部（N-02）の**2件のみ**で、いずれもCould/Phase3の周辺価値であり、v2の中核判断（TAB必須化・テンポマップMust・整譜品質群・下書きUX・完全ローカル/成果物非関与）を揺るがさない。

残る懸念は3点、いずれも軽微:
1. **移動ド記譜の欠落（N-01）** — 耳コピ学習層への訴求で拾える小さな穴。F-064と束ねてCould追加を推奨。
2. **合唱/アカペラ声部の扱い（N-02）** — 有償人的市場が実在。F-019/F-020の受入条件に声楽小編成を明記する程度の対応で十分。
3. **简谱の位置づけ** — 中華圏では第一級出力。F-033据え置きは妥当だが、中華圏本格展開時は五線と同格の再検討余地（実装フェーズ判断）。

v2改訂は前回6発見を過不足なく吸収し、降格判断（捨てる勇気）も第2巡の証拠で追認された。要件網羅性は実務上の飽和点に達していると判断する。
