## 調査サマリ

X上で強い実需は次の5点です。

1. **ステム分離 → コード/MIDI → 下書き譜**
2. **ギターTABの実演奏可能性（把位・指法）**
3. **简谱（数字譜）**
4. **ローカル／無料／非アップロード**
5. **フルミックス多楽器 → MIDI**（2026-07 MuScriptor周辺で急増）

コア要件（F-003/010/014/021/031/032/034/038/066 等）は概ねカバー済みです。ギャップは「周辺導線」と「優先度の歪み」に集中しています。

### 4.2 ヒット状況（系統別）

| 系統 | ヒット質 | メモ |
|---|---|---|
| 中国語 扒谱 / AI扒谱 | **中〜良** | 需要・自作・OSS紹介が混在。宣伝（AudioJam等）も多い |
| 中国語 简谱 | **良** | 「听歌生成简谱AI求推荐」など**欠落に直結** |
| 中国語 吉他谱/把位 | **良** | TAB依頼、Songsterr+動画修正WF |
| 中国語 钢琴谱 | **中** | 手扒完了報告が多く、ツール不満は少なめ |
| 中国語 网易云 曲谱 | **弱** | 機能そのものへの皮肉/言及。AI統合の「方向性」コメント1件 |
| 中国語 编曲×扒谱 | **中** | 学習意欲・コード理解不足 |
| 英語 audio-to-MIDI 精度 | **良** | 「何年も不正確」「AbletonもLogicもbad」 |
| 英語 chord detection / play-along | **良** | 自作アプリが強い |
| 英語 multi-instrument (MuScriptor) | **非常に強（2026-07）** | ただし研究/リリース反応が中心 |
| 英語 Songsterr / AI tab 品質 | **良** | slop / wrong の生の不満 |
| 英語 drum transcription | **中** | サービス提供者投稿が多く、需要は「有料で買う」形 |
| 英語 worship/gig/music director charts | **弱〜ゼロ** | クエリが宗教/チャート単語に吸われ、**礼拝バンド現場の採譜需要はXではほぼ不検出** |
| 英語 lick / solo transcription | **弱** | 散発 |
| 英語 offline privacy（音楽採譜限定） | **中** | 会議transcriptionのlocalが大半；音楽はStemDeck/MuScriptor経由で間接ヒット |

### 4.3 調べても見つからなかった／弱かった領域

1. **Worship team / music director の現場チャート需要** — 指定系統だがX上は宗教・金融「chart」ノイズが多く、**礼拝バンドのリードシート実務はほぼ観測不能**。Reddit/Facebookグループの可能性大。  
2. **Gig transcribing 専門職** — 同様にヒットせず。  
3. **Lick transcription 専用ツール要望** — ほぼなし。  
4. **Klangio/AnthemScore個別の課金炎上** — 今回のXサンプルでは薄い（既存Trustpilot/YouTube依存のまま）。  
5. **sargam / インド古典** — 完全ゼロ。  
6. **変則チューニング「機能として欲しい」** — Drop Dの演奏言及はあるが、採譜ソフト要望としては弱い（既存「最多要望」はWeb競合調査由来）。  
7. **日本語圏** — 本タスク範囲外だが、X日本語の扒谱相当は「耳コピ」語彙で別スキャンが必要。  
8. **中国語の「太难/求助」** — 「扒谱」が比喩（データ扒取等）に吸われ、**音楽限定の精度が落ちる**。音楽文脈は「简谱」「吉他谱」「AI扒谱」の方が純度が高い。

### 4.4 方法論上の限界（事実として記載）

- Xの中国語音楽ユーザー密度はB站より低い可能性。  
- 英語の実務ギタリスト/礼拝ミュージシャンはInstagram/Facebook寄り。  
- セマンティック検索はAI音楽生成・stem分離ツール宣伝を多く返す。  
- したがって本リストの「需要ゼロ」は **「X両言語で観測されなかった」** であり、市場ゼロではない。
