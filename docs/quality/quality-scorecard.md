# 品質採点表（100点満点）

**制定:** 2026-07-20（ユーザーゴール「100点にしてください」の運用定義・Issue #53）
**原則:** 100点 = コア8件(C1〜C8)の受入条件(core-requirements-v3.md §3)が全て自動テスト/ベンチで実装され、グリーンで、数値目標を満たした状態。配点は受入条件の逐条分解。スコアは実測のみ（推測記入禁止・function-critique-protocol.md準拠）。
**スコープ外（ユーザー裁定でユーザーテスト段階に封緘済みの項目）:** 受入デモ3種の実施（ノック/雨+口ずさみ/エアコンhum）・MuseScore等実機インポート確認・人間聴感との相関キャリブレーション。これらは本採点表の分母に含めない（採点表100点=「ユーザーテスト前に自動で到達できる完成」）。

## 補足（正直な記録）

- C1-1 の「素点0.709」は normal感度のBP生イベント平均（第1回ベンチ）。#38のraw 0.678はhigh感度構成での測定であり、受入条件の分母(0.709)には未達。密度適応(#54)で両立を狙う。
- 2週連続の非劣化維持（完了の定義 §5）は時間経過を要するため本採点表の対象外とし、達成後に §5 を別途宣言する。
- 採点表の更新は実測を伴うコミットでのみ行う。
