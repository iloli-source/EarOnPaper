## 1. 修正の頑健性検査

### notate 弱起処理の変異(score.py:254 `_drop_leading_silence`)

| 攻撃 | 入力 | 実挙動 | 判定 |
|---|---|---|---|
| 小節内弱起 | 先頭音 beat2.5開始 | 先頭Rest(2.5)を保持・巨大休符なし・3小節 | OK |
| 複数小節先頭無音 | 先頭音 beat10(2.5小節) | 完全2小節シフト→残り2拍を小節内休符化 | OK |
| 途中の長無音 | 4音→8拍空白→4音 | 先頭は音符・ギャップは休符2個に統合・崩壊なし | OK |
| 交差メロディ | 低音↔高音を拍毎 | 純高さで機械分割・両手とも半分休符(R1のP1が残存・既知) | P1(既知) |

弱起の巨大休符(R1のP1)は**完全に解消**。`lead_measures = int(first // BEATS_PER_MEASURE)` の完全小節単位シフトが小節内弱起を壊さない設計で、変異入力に頑健。

### engrave 豆腐ゼロの他テンポ/特殊値耐性(engrave.py:56 `plain_tempo_svg`)

BPM値 60 / 63.5 / 88 / 120 / 144 / 208 / 30.25 の全てで tempo ブロックの music-font tspan が0・"BPM <数値>"表記に変換・豆腐残存ゼロ。**小数BPM・極端値でも保つ**。譜面本体のグリフ(符頭・音部記号)は`<use>`参照方式(SVG内19個)でtspanテキストではないため豆腐化しない(検証済み: SMuFL tspanは全SVGで1個=テンポのみ、修正後0個)。豆腐ゼロは頑健。

### quality 審判v2の騙し攻撃 変異版(ears.py register/health/score_rhythm)

ground-truth = きらきら星系30音メロディ(BPM100)。baseline(同一)= overall **0.9867**(v2重み)。

| 攻撃 | overall(参照なし) | register | offset | 判定 |
|---|---|---|---|---|
| baseline(+0) | 0.9867 高一致 | 1.0 | -0.3 | — |
| **+7半音移調** | **0.8278 高一致** | 0.892 | 6.7 | P2 |
| +12オクターブ | **0.8652 高一致** | 0.058 | 11.7 | **P1** |
| **50%オクターブ上げ(交互)** | **0.9856 高一致** | **1.0** | 4.7 | **P0** |
| register騙し(長anchor+半分誤り) | **0.9832 高一致** | **1.0** | -4.3 | **P0** |

**新規発見(P0): register指標は中央値比較ゆえ部分オクターブ誤りを検出不能。**
`register_match`(ears.py:190)は音価加重**中央値**の比較。直接プローブで、
40%の音をオクターブ上げても score=1.0・offset=2.7(検出ゼロ)。50%交互でも
median不動でoverall 0.986「高一致」。「長い正register anchor音を1本足す」だけで
全音+12でも中央値が引き戻され score=1.0 に戻る(register騙し攻撃)。
最頻の実務誤り(部分的オクターブ飛び)を審判はすり抜けさせる。R1のP0(全音+12)
は塞いだが、**部分オクターブ誤りという同型の穴が残存**。

**ダッシュボード数字の不一致(P1): 「+12オクターブ 0.90→0.762」は本番経路で再現不可。**
参照なし(=本番運用)経路の実測は **0.8652「高一致」**(chroma0.99×0.35 + onset0.98×0.35
がregister0.058×0.125を圧倒)。offsetは11.7と正しく表示されるが、verdictは
「高一致 — 手直しは軽い可能性」のまま=丸ごとオクターブ違いを「軽い手直し」と誤標識。
ダッシュボードの0.762はこの条件では出ない(別メロディか中間重みでの測定と推定)。

## 2. ダッシュボード限界の境界確認(定量化)

**転調追従なしの実害度(定量)**: `turkish_real.musicxml` を再解剖 —
key=Aメジャー(3♯)・1957音中 displayed accidental **261個=13.3%**。
全12長調で総当たりした結果 **A長調が最小(13.5%)**・C長調なら22.4%。
→ **ダッシュボードの「Aメジャー調号に臨時記号だらけ」は過大表現**。realは
非転調曲で、13.3%は単一大域調の**最適値**(採譜のピッチ雑音由来であって転調由来ではない)。
真の実害は**転調曲でのみ**発生: 合成C→E転調曲(各16小節)で単一大域調(C,0♯)は
64音中accidental **16個=25.0%**、E半分に全部集中。区間調なら~0%。
→ 限界は実在するが「転調素材でのみ25%コスト。非転調realでは追従不要」と台帳訂正すべき。

**密集小節の窮屈さ(定量)**: 16分音符64個・両手アクティブ4小節をPNG化(視覚確認)。
2段組・拍/連桁正常・重なりゼロ・豆腐ゼロ(Eb調号)。Verovio既定間隔が16分密度を
捌ける。**「窮屈さ」はP級欠陥ではない**(Verovioレイアウトが健全)。
