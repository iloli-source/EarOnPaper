# ADR-004: 譜面描画（エングレービング）エンジンの選定

**日付:** 2026-07-19
**状態:** 採択
**関連:** Issue #41（ユーザーテスト解禁の前提条件=五線譜PDF出力）/ NF-026（コピーレフト分離）/ NF-029（ライセンス台帳）/ NF-001（オフライン動作）

## 候補比較

| 候補 | ライセンス | 導入 | オフライン | 組込性 | 品質 | 判定 |
|---|---|---|---|---|---|---|
| **Verovio 6.2.1** | **LGPL-3.0**（※Issue記載のMITは誤り。実確認でLGPL） | pip 1発（C++コアのPythonバインディング） | ✅完全 | ◎ ライブラリとしてtoolkit APIを直接呼べる。JS版もありアプリ組込の将来性が高い | ○（MEIベース・実用十分） | **採択** |
| LilyPond | GPLv3 | brew（重量級・依存多） | ✅ | △ 外部プロセス起動のみ（プロセス境界でGPL隔離は可能） | ◎（最高品質） | 将来の高品質印刷オプション候補として温存 |
| MuseScore CLI | GPLv3 | アプリ本体（~GB）必須 | ✅ | △ 外部プロセス | ○ | 不採用（導入が重く、競合製品への依存はG3法務観点でも避けたい） |

## 決定

**Verovio を採択。** MusicXML→SVG は Verovio toolkit、SVG→PDF は cairosvg（LGPL-3.0）、複数ページ結合は pypdf（BSD）で行う。

- パイプライン: MusicXML → verovio.toolkit（ページ毎SVG） → cairosvg（ページ毎PDF） → pypdf（結合）
- すべて pip 導入・完全オフライン・端末内完結（成果物非関与構造と整合）

## ライセンス整理（NF-026/029）

- Verovio・cairosvg は **LGPL-3.0**: 動的リンク相当（Pythonモジュールとしての利用）であり、自社コードへのコピーレフト波及はない。**配布時**はLGPL条文に基づく告知・ライブラリ差し替え可能性の担保が必要 → ライセンス台帳（NF-029）に登録し、リリース前パッケージングでの最終確認事項とする（G3法務パッケージに追記不要・技術対応で完結する性質）
- GPL（LilyPond/MuseScore）は**プロセス境界の外**でのみ使用可という既存原則を維持

## 帰結

- services/notate に engrave 段（MusicXML→PDF/SVG/PNG）を追加。pipeline に `--pdf` オプション
- これにより「音源→五線譜PDF」の一気通貫が成立し、**ユーザーテスト段階の前提条件が充足**される
