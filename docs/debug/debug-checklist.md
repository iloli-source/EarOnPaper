# デバッグ観点チェックリスト（再利用）

**由来:** 外部デバッグ EOP-DEBUG-20260721-001（21項目・[調査ログ](EOP-DEBUG-20260721-001.md)）を一般化した観点集。
**用途:** 以降のデバッグ/テスト設計で毎回この観点を通す。合成テストが全緑でも、下記の観点が欠けていると実バグが残る（今回はまさに「358件全緑」の裏で21件が残っていた）。
**適用方針:** 新規モジュール実装時・レビュー時・回帰テスト設計時に、該当する観点の回帰テストを追加する。純関数は境界値、外部境界(IPC/CLI/ファイル/OS)は信頼境界・クロスプラットフォーム・ライフサイクルを重点確認。

---

## A. 境界値・敵対入力（数値・サイズ）

- [ ] **非有限値**: すべての数値経路に `NaN / +Infinity / -Infinity` を通す。`int(round(inf))` は OverflowError。`not (x > 0.0)` は +Inf を**通してしまう**非対称ガード → `math.isfinite()` を先に確認。（3.12）
- [ ] **巨大有限値**: 出力が入力に比例して増幅しないか（`" -" * n` 等）。上限クランプで有界化。（3.13）
- [ ] **入力比例の反復ループ = DoS**: 1ステップずつの `while` は巨大入力(±10^12級)で停止しない → 必要回数を**算術で一括算出**し定数時間化。（3.11）
- [ ] 空入力・極小長・全ゼロ・負値の安全な既定。

## B. クロスプラットフォーム

- [ ] **パス区切り**: `split('/')` はWindowsの `\` を落とす → `\\` と `/` の両対応 basename。（3.3）
- [ ] **OS別実行ファイル**: `.venv/bin/python`(POSIX) vs `.venv\Scripts\python.exe`(Win) を候補化し**実在解決**。固定参照は環境で起動前失敗。（3.2/3.1/3.6）
- [ ] **file:// URL**: 空白・`#`・`?`・非ASCII・ドライブ表記を `pathToFileURL()` で符号化（手組み `file://${p}` は破綻）。（3.4）
- [ ] 絶対/相対パスの解決基準（cwd依存に注意。研究/設定ファイルの相対パスは実行ディレクトリでズレる）。

## C. 偽成功・完了条件

- [ ] **成果物の実体検証**: 終了コード0＋JSON解析成功だけで成功応答しない。生成ファイルが**存在し非空**であることを確認してから resolve。（3.8）
- [ ] **外部API戻り値**: 例外でなく**エラー文字列/コードを返す**API（Electron `shell.openPath` 等）は戻り値を確認。（3.9）
- [ ] 「テスト通過 ≠ 本番動作」。実データ読み返し・実行検証を完了条件に含める。

## D. 信頼境界・セキュリティ

- [ ] **境界での再検証**: IPC/CLI/外部から来た入力は境界で再検証。**UIフィルタを検証と混同しない**（allowlist + `isFile()`）。（3.16）
- [ ] **パス包含判定**: 文字列prefixは偽装可（`/job` が `/job-evil` を誤許可）→ `path.relative()` で `..`開始・絶対を拒否。（3.20）
- [ ] **任意ファイル操作の遮断**: コピー/外部オープンは**アプリ管理下の生成物**に限定。（3.14/3.15）
- [ ] 秘密情報・音源内容をログ/診断に含めない。

## E. リソース・ライフサイクル

- [ ] **一時ディレクトリ**: 生成物を保持する設計なら**終了時に再帰削除**（`before-quit` 等）。放置は容量消費＋情報残留。（3.17）
- [ ] **子プロセス**: `spawn` したプロセスを追跡し、アプリ終了時に **kill**。（3.18）
- [ ] **同期I/Oでイベントループを止めない**: `copyFileSync` → `await fs.promises.copyFile`。（3.10）
- [ ] **入力上限**: title等の外部文字列は上限クランプ（引数/ログ/メタ肥大化防止）。（3.19）

## F. テスト性能・停止性

- [ ] **O(N×M)を疑う**: 直接畳み込み `np.convolve` → `scipy.signal.fftconvolve`。テストのTIMEOUTはアルゴリズム/データ規模の兆候。（3.7）
- [ ] 重い実データ依存テストはskip-if-missing＋合成での高速回帰を用意。

## G. 静的品質

- [ ] **未使用変数・import**（Ruff/lint 通過）。意図不明コードは残さない。（3.21）

## H. 機能接続（wiring）

- [ ] **ログ/イベントがUIに実接続されているか**: 購読が `() => {}` で破棄されていないか。タイマー任せの見かけ進捗になっていないか。（3.5）
- [ ] 新規モジュールが `__init__`/pipeline へ配線され、実際に呼ばれる経路があるか。

---

## メタ教訓

- 合成ユニットテストは「正常系の設計どおり」を確認するが、**境界値・クロスプラットフォーム・信頼境界・ライフサイクル・偽成功**は観点として意図的に足さないと抜ける。
- 「◯◯件全緑」を安全の根拠にしない。**このチェックリストの各観点で回帰テストを1つ以上追加**してから完了とみなす。
- 修正時は必ず**再現テスト（RED）→修正（GREEN）**で、部分結果を正式証跡に流用しない。
