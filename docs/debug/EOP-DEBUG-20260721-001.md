## 3.2 ElectronのPython実行パスがPOSIX固定

- **分類**: クロスプラットフォーム／起動不能
- **重大度**: 高
- **対象**: `app/main.js`, `app/platform-utils.js`
- **検出方法**: ソース静的監査＋Node単体テスト
- **修正前**: `path.join(ENGINE_DIR, '.venv/bin/python')` を全OSで使用。
- **原因**: Windows仮想環境の実行ファイルは通常 `.venv\Scripts\python.exe` であり、元実装はWindows候補を持っていなかった。環境変数による明示指定やPATH上のフォールバックもなかった。
- **影響**: Windowsで採譜処理がPython起動前に失敗する可能性が高い。仮想環境構成差にも弱い。
- **修正**: `executableCandidates()` を新設。Windowsは `Scripts/python.exe`、macOS/Linuxは `bin/python` 系、環境変数指定、PATHコマンドを候補化。`resolveExecutable()` で実在パスを優先し、コマンド名へフォールバック。
- **追加調査**: 局所Nodeテスト初回はWindows候補生成が1件失敗。ホストOS依存の `path.join` を使用していた接続不備を確認し、`path.win32.join` / `path.posix.join` の明示分岐へ修正。
- **再検証**: Node 10/10 PASS。追加検証で `js_python_runtime_candidates` がPASS。
- **判定**: FIXED
