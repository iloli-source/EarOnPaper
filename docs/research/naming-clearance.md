# AI採譜プロジェクト 命名クリアランス簡易調査

作成日: 2026-07-18
調査手法: WebSearch / WebFetch による公開情報の簡易確認
対象: プロダクト名 / リポジトリ名の候補について、商号・商標・既存サービス・ドメイン・アプリの衝突を確認

> **重要な前提（免責）**
> 本調査は公開Web検索とサイトアクセスによる**簡易クリアランス**であり、法的な商標調査ではありません。
> J-PlatPat（特許庁）は動的サイトでプログラムからの網羅検索が困難なため、本調査では商標登録の有無を
> **Web検索の結果と一般的な商標法の原則から推定**しています。正式な採用・出願前に、必ず**弁理士による
> 正式な商標調査（J-PlatPat / USPTO の指定区分・類似群コードでの検索）**を実施してください。
> 特に音楽ソフト／サービスの主要区分は **第9類（ソフトウェア）・第41類（教育・エンタメ）・第42類（SaaS）** です。

## 各候補の詳細

### 1. Playing by ear / playing-by-ear — 衝突リスク: 中

- **既存サービス**: 「playing by ear」「by ear」は音楽採譜業界の**一般的な言い回し**。
  Transcribe a Song（「100% by ear」）、Notable Score（「transcribing music by ear」）など、
  複数の採譜サービスがキャッチコピーとして常用している。特定企業の独占的表示ではない。
- **商標**: 慣用的なフレーズであり、音楽・採譜分野では**識別力が弱く商標化が難しい**可能性が高い。
  同分野での独占は困難と推定。
- **ドメイン**: `playingbyear.com` は **Abbott家の個人ポータルサイト**として実在・稼働中（Strum Machine等へのリンク集）。
  .com取得は不可。`.jp` / `.ai` は未確認だが、ブランドとして.comが押さえられている点は不利。
- **アプリ**: 直接の同名アプリは未確認だが、「ear」系アプリが乱立（下記3参照）。
- **判定根拠**: 語感は良いが、(1)業界の一般表現で差別化しにくい、(2)識別力が弱く商標防御力が低い、
  (3)主要ドメインが他者保有。**中リスク**。

### 2. 耳コピー / 耳コピ / mimicopy / ミミコピ — 衝突リスク: 高

- **既存サービス（直接衝突）**:
  - **mimiCopy**（ART Teknika Inc. 開発）— iOS向け耳コピ専用プレーヤーの**有名アプリ**。App Store配信中、公式サイト `mimicopy.artteknika.com`。
  - **mimi-copy.com** — 楽譜作成・耳コピ代行の**採譜業者**（別事業者）。
  - 「耳コピ」は各種メディア（島村楽器 Digiland 等）で**一般名詞として定着**。
- **商標 / 普通名称**: 「耳コピ」は音楽制作の**普通名称（識別力なし）**に該当する可能性が極めて高く、
  商標法第3条1項の登録拒絶事由（普通名称）に当たると推定。仮に登録できても防御力は低い。
- **ドメイン**: `mimi-copy.com` は採譜業者が保有。`mimicopy` 系は既に使われている。
- **アプリ**: mimiCopy が App Store で確立。同名は明確に不可。
- **判定根拠**: 既存有名アプリ＋採譜業者と**名称が完全一致**し、かつ普通名称で識別力もない。
  **高リスク。採用非推奨。**

### 3. earcopy / ear-copy — 衝突リスク: 高

- **既存アプリ（直接衝突）**:
  - **EarCopy - Music Practice**（App Store, id6759784749）— 移調・ABループ・EQ搭載の**採譜/練習アプリ**。まさに同ジャンル。
  - **Simple Ear Copy**（App Store, id6759311175）— 速度/ピッチ変更・EQ搭載の練習プレーヤー。
- **一般名称性**: 「ear copy」は「耳コピ」の英訳的表現で、音楽練習分野の記述的用語。識別力は弱い。
- **ドメイン**: `earcopy.com` は**アクセス時に接続拒否（ECONNREFUSED）**。稼働サイトなしだが取得状況は要確認（DNSは存在の可能性）。
- **判定根拠**: **同ジャンルの同名アプリが複数存在**（しかも新しめのアプリ）。真っ向から被る。
  **高リスク。採用非推奨。**

### 4. saifu / saifu-ai（現リポジトリ名） — 衝突リスク: 中

- **既存サービス（直接衝突）**:
  - **Saifu**（チェコ・プラハ、2016年設立）— **暗号資産ウォレット＋決済プラットフォーム**。`saifu.ai` は稼働中
    （"A new revolutionary Saifu wallet is coming soon"）。Crunchbase/Tracxn に企業掲載あり。
  - 中国 Fuzhou Saifu 社（電子/交通製品）など同名企業も複数。
- **意味の問題**: 「saifu（財布）」は**日本語の一般名詞**で、意味は「wallet」。
  **音楽・採譜と全く無関係**であり、ブランドの意味的整合性がない。むしろ金融/ウォレット連想が強い。
- **商標**: 一般名詞のため、金融分野では暗号資産Saifuが先行。音楽分野で新規取得の余地はあるが、
  「財布」という語自体が識別力を持たせにくい。
- **ドメイン**: `saifu.ai` は暗号資産Saifuが保有。プロダクトの外部ブランドとしては使いにくい。
- **判定根拠**: 外部公開プロダクト名としては(1)暗号資産Saifuとの連想衝突、(2)音楽と無関係な意味、で**中〜やや高**。
  ただし**社内リポジトリ／コードネームとしてなら実害は小さい**ので温存は可。**公開プロダクト名には非推奨。**

## 提案する代替名（衝突が少なそうな3案）

> いずれも「AI採譜／耳コピ」の語感を残しつつ、既存サービスとの直接衝突を避けた案。
> 造語・和名寄りにすることで**識別力（商標登録の通りやすさ）**を確保する狙い。

### 提案A. Otohiki（音引き / otohiki） — 衝突リスク: 低 ★第1推奨

- **語源**: 「音（oto）」＋「引き（採る・写し取るの語感）」。採譜＝音を写し取る、を想起させる和製造語。
- **衝突確認**: Web検索で音楽アプリ・企業・サービスとしての**目立つ衝突なし**。同名App/Playアプリも未検出。
- **利点**: (1)造語に近く識別力が高い、(2)日本語圏で採譜の意味が直感的、(3)ローマ字表記でグローバルにも通る。
- **要確認**: `otohiki.com` / `.jp` / `.ai` の空き、J-PlatPatでの「音引き」商標、Play/App Store。
- **判定根拠**: 現時点で明確な衝突が見当たらず、意味整合性・識別力ともに良好。**低リスク。**

### 提案B. Tonebind（tonebind） — 衝突リスク: 低 ★第2推奨

- **語源**: 「Tone（音）」＋「bind（束ねる／楽譜にまとめる）」の英語造語。
- **衝突確認**: Web検索で音楽アプリ・企業としての**結果ヒットなし**（未使用の造語）。
- **利点**: (1)完全な造語で商標化しやすい、(2)英語圏でも発音しやすくグローバル展開向き、(3).com取得の可能性あり。
- **注意**: 「-bind」系のSaaS名は他分野に散見されるため、指定区分（第9/41/42類）での確認は必要。
- **要確認**: `tonebind.com` / `.ai` の空き、USPTO/J-PlatPat。
- **判定根拠**: 造語ゆえ衝突が最も少なく防御力も高い。ただし音楽・採譜の意味は説明が要る。**低リスク。**

### 提案C. Sonorai / Kikitori 系 — 衝突リスク: 中（参考案）

- **Sonorai**（"sonor"＝響き 由来の造語）: 検索すると **Apple Music / Spotify に「Sonorai」という音楽アーティスト**が存在。
  音楽分野での同名がすでにある点が不利。→ 中リスク。
- **Kikitori（聞き取り）**: **Google Play にデベロッパー名 "kikitori" が実在**。加えて「kikitori（聞き取り）」は
  日本語学習分野で「リスニング」を指す一般語として定着。識別力・衝突の両面でやや弱い。→ 中リスク。
- **判定根拠**: どちらも語感は良いが既存用途があるため、AとBを優先。採用検討時は要追加確認。

## 音楽採譜市場の混雑状況（参考）

採譜／楽譜化ソフトのカテゴリは**すでに強力な既存プレーヤーが多数**存在する。命名時は下記との被りも避けること:

- ScoreCloud / AnthemScore / Songscription（AI） / Klangio（Transcription Studio） / Soundslice /
  Transcribe!（Seventh String） / Tabify（GitHub） / Notable Score / Transcribe a Song 等。

「score」「note」「transcribe」「stave」「tab」を含む名前は競合と被りやすく、記述的で識別力も弱いため注意。

## 総括と次アクション

1. **公開プロダクト名の推奨**: **Otohiki**（第1）または **Tonebind**（第2）。
   現在の `saifu` / `saifu-ai` は**社内コードネーム／リポジトリ名としては温存可**だが、外部ブランドには非推奨。
2. **採用非推奨**: 「耳コピ / mimicopy」「earcopy / ear-copy」は既存アプリ・業者と直接衝突するため回避。
3. **正式出願前に必須**:
   - 弁理士による **J-PlatPat 正式調査**（第9類ソフトウェア／第41類教育・エンタメ／第42類SaaS＋類似群コード）。
   - **USPTO / EUIPO** の同区分検索（海外展開を見込む場合）。
   - 主要ドメイン（`.com` / `.jp` / `.ai`）と App Store / Google Play の**最終空き確認**。
   - 選定名の**普通名称・記述性**の再チェック（識別力が登録可否を左右する）。
