## 総合判定

## 4モデル共通の指摘（信頼度: 高）

1. **「OSS空白＝勝ち筋」は事実誤認または論理の飛躍**
   - Songscription が Arrangement Mode・難易度レベリング・人手ハイブリッド代行を**実装済み**。2025年11月に **$5M調達（Reach Capital主導）、15万ユーザー・150カ国・月次60%成長**（Opus裏取り: MusicBusinessWorldwide/Billboard）
   - Klangio（2018年創業・独）も Arrangement Mode / Lead Sheet Mode / 難易度2段階を提供済み
   - OSSに整譜がないのは「儲からない/論文にならない/プロダクト開発の泥仕事だから」であり、空白は機会ではなく警告
   - MuScriptor は重みが **CC-BY-NC（商用利用不可）**。「OSSで解決済み」も「商用に使える」も両方誤り

2. **実機検証ゼロが全結論を無効化**
   - 「5分の無料トライアルで潰せた誤り（=空白仮説）をデスクリサーチで勝ち筋に格上げした」（Opus）
   - 「実用域」の定義が未定義。frame/note F1 と「読めて弾ける楽譜」の混同（grok: 定義の詐術）

3. **X調査は生存者バイアス＋宣伝混入**
   - MuScriptor公開直後のwow報告が過大に反映。失敗談は Reddit/フォーラム/YouTubeコメントに偏在し未調査

4. **ココナラ「5.1万件・2,000〜3,000円」の読み方が雑**
   - 5.1万件はカテゴリ**累計**であり年間フローではない。手数料22〜27.5%控除後の手取りと作業時間から**時給換算で最低賃金割れ**＝「割に合わない労働が薄利で回っている」証拠
   - Sonnet試算: 採譜代行TAMは最大でも年25億円規模。Sheet Music市場全体でも約$375M・CAGR 1.11%の低成長市場

5. **著作権リスクの過小評価（最重要級）**
   - **編曲権はJASRAC管理外**。Piascoreの包括許諾は「そのままの楽譜」にのみ有効で、**Arrangement Mode（難易度別アレンジ・左右手再構成・簡略化）は法的に全て「編曲」＝1曲ごとに音楽出版社の個別許諾が必要**
   - **ロクラクII最高裁判決**の複製主体論により、SaaSがサーバ側で複製・変換する形態は「ユーザーの私的複製」と認められない可能性が高い（Sonnet）
   - 「儲けようとして編曲・販売に踏み込むほど権利処理が指数的に重くなり、私的複製ツールに留めるほど課金理由が消える」ジレンマ（Opus）

6. **アーキテクチャ議論はYAGNI違反**
   - PMF前にDDD/クリーンアーキ/モジュラーモノリスを設計するのは「負ける前提の城の石垣を丁寧に積む」行為。検証すべきは「整譜が競合より上手くできるか」の一点

## 批判から抽出した要検証課題（→ GitHub Issue化）

1. 競合実機ベンチ: Songscription/Klangio/AnthemScore に実曲を通し整譜品質を採点（「空白」仮説の実機検証）
2. 編曲権の法務整理: 提供形態別（ツール/代行/販売）の権利処理要否マトリクス作成
3. ユニットエコノミクス試算: ハイブリッド代行の実コスト・実単価・利益の数値化
4. MuScriptorライセンス影響確認: CC-BY-NC が各事業形態に及ぼす制約
5. 「実用域」の定量定義: 評価データセットと採点関数の設計（EDD基盤）
6. 市場規模の実測: 採譜のTAM/年間フロー/成長率の一次データ収集
7. 生存者バイアス補正: Reddit/フォーラム/YouTubeコメントの失敗談の系統収集
