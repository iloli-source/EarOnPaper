**投資委員会レビュー結論**

対象: [conclusions-v4.md](REPO_ROOT/docs/research/conclusions-v4.md) / [round3 critique-summary.md](REPO_ROOT/docs/research/critique/round3/critique-summary.md)

辛口に言うと、プランB採択は許容。ただし v4 は「検証計画」としては前進しているが、「投資判断に耐えるプロトコル」としてはまだ甘い。特に G0 を条件付き通過扱いしている点と、G1 の評価設計を MIR 標準と呼べるかの扱いが危ない。

## 1. ベンチ設計はMIR標準と整合するか

**一部だけ整合。だが、v4のままだと“学術標準に寄せた風の事業ベンチ”で止まる。**

`mir_eval`併記は妥当。`mir_eval.transcription`は、音符を onset / offset / pitch の離散イベントとして扱い、precision / recall / F-measure / overlap を計算する、MIREX由来の低レベルAMT評価に沿うものだから。公式ドキュメントも「最も単純で一般的な指標」に限定していると明記している。つまり、**音高・オンセット検出の足切り指標としては正しい**。ただしこれは譜面品質そのものではない。`mir_eval`自身も piano-roll 的な音符列評価で、声部、拍節、和声、可読性までは見ない。

ここで v4 の弱点が出る。採譜プロジェクトの勝負所は「MusicXMLとして人間が読める譜面」なのに、学術側ではその不足を埋めるために MV2H や MUSTER が出ている。MV2H は、従来評価が multi-pitch / MIDI寄りで、voicing・metrical structure・harmony の評価が不足していることを問題にして導入された指標。 MUSTER も MusicXML同士を比較し、譜面の複数側面を edit distance 系で測る score transcription 用指標である。

したがって、v4の「mir_eval併記」は正しいが、**MV2H/MUSTERを入れずに“整譜品質を客観指標で見る”と言うなら誇大**。最低限、G1はこう直すべき。

- `mir_eval`: 音高・onset・offsetの低レベルAMT診断
- `MV2H`または`MUSTER`: MusicXML出力がある対象の譜面構造診断
- 人手手直し時間: 実用価値・経済価値の主指標
- 2名ブラインド: 学術標準ではなく、事業ベンチ上のバイアス低減策
- 3帯判定: MIR標準ではなく、投資判断用の事前意思決定ルール

測定者2名ブラインドは良いが、学術標準とは言えない。2名では信頼性推定が弱く、少なくとも「不一致時の裁定者」「一致率」「帯判定の再測ルール」は必要。ツール名を伏せるだけでも、譜面の癖・出力形式・レイアウトでバレる可能性がある。

## 2. 弁護士3問は答えが出る形か

**方向は良い。ただし、今の3問はまだ抽象度が高く、弁護士が“場合によります”で返しやすい。**

文化庁の平成27年クラウド報告書は、タイプ2を「プライベート・ユーザーアップロード型」と整理し、その中に「変換機能付加型」を含めている。([bunka.go.jp](https://www.bunka.go.jp/seisaku/bunkashingikai/chosakuken/hogoriyo/h26_10/pdf/shiryo_1.pdf)) ただしタイプ2の許諾不要整理は、あくまで個別サービスの機能・提供態様を全体で見て判断するという留保付き。 一方、メディア変換サービスは、自炊代行判決を前提に、事業者が複製主体とされ私的使用目的の複製に当たらない方向で整理されている。

つまり Q1「タイプ2変換型該当性」は良い問いだが、**“採譜は保存でなく変換”だけでは足りない**。採譜は単なるファイル形式変換ではなく、音源から音楽著作物の表現を楽譜として再現する行為なので、複製権だけでなく翻案・編曲的評価、原曲の創作的表現の再製性、レコード/実演隣接権の処理も絡む。

Q2「私的使用該当性」も良いが、SNS配信・イベント演奏・バンド練習を一問に詰めるのは雑。30条は「個人的に又は家庭内その他これに準ずる限られた範囲」で、使用者自身による複製が軸。用途別に回答を分けるべき。バンド練習とSNS公開と有償イベント演奏は、同じ箱に入れてはいけない。

Q3「30条の4ただし書」は必要。ただし、30条の4は非享受目的利用の話で、ユーザーに譜面を返す推論サービスそのものを正当化する条文ではない。文化庁のAI整理も、法的拘束力はなく、具体技術の確定評価ではないと断っている。 さらに、享受目的併存やただし書該当で30条の4が使えない場合、複製は侵害となり、差止・損害賠償等のリスクが出る。

弁護士への質問は、3問ではなく「3問＋事実別表」にすべき。入力音源、処理場所、保存期間、出力形式、ユーザー用途、共有機能、人手介在、モデル更新、学習データ出所、JASRAC/出版社/原盤権者への許諾要否を表にして渡さないと、実務回答にならない。

## 3. G0の「後処理3-6人月」は妥当か

**かなり楽観。3-6人月は“1ジャンル1症状の実証スパイク”の上限であって、競合超えプロダクトの工数ではない。**

Basic Pitch は公式に audio-to-MIDI で、出力もMIDI・raw model output・note eventsが中心。MusicXMLの整譜エンジンではない。([github.com](https://github.com/spotify/basic-pitch)) その後段を作るには、MIDIから譜面へ変換する泥仕事が必要になる。

MuseScoreのMIDI import公式仕様を見ると、後処理で扱う対象は、quantization、最大声部数、tuplets、human performance用 beat tracking、小節位置、拍子、左右手分割、clef changes、duration simplification、dotted notes、pickup、swing まで広い。 これは「ちょっとルールを書く」レベルではない。v4が狙うリズム/拍子/整譜の崩れ方そのものが、既存記譜ソフトで長年UI付き調整対象になっている。

研究側でも、audio-to-score piano transcription は note-level で高精度でも、tempo scale、meter、bar line positions の誤りが残ると報告されている。 つまり「音は取れているが譜面が崩れる」は、まさに3-6人月で簡単に埋まる周辺バグではなく、問題の本体。

G0はこう書き換えるべき。

- `3-6人月で越えられる`: 固定テンポ・単一楽器・拍子既知・小節線既知・表情揺れ小の限定ケースのみ
- `3-6人月で越えられない`: 拍子推定、downbeat推定、声部分離、左右手分割、tuplets/swing、ポップス複数楽器、表情演奏、読みやすいMusicXML
- `3人月スパイク成功`: 競合超えではなく、1ジャンルで「崩れの1主因が規則後処理で減る」ことの証明に限定

今の「最良競合より20pt改善」は、n=9やn=25-30では測定ノイズと選曲依存が大きすぎる。平均で勝ったと言えても、商品として勝てるとは限らない。

## 4. v4に残る事実誤り・危険な記述

- **Basic Pitchの位置づけが揺れている。** §1ではAMT直接競合集合に入れ、§5では参照値に落としている。公式には audio-to-MIDI であり、音声→譜面の直接競合というよりOSS下限/素材供給源。([github.com](https://github.com/spotify/basic-pitch))

- **Klangioは“参照すべき直接競合”としてもっと重い。** 公式に multi-instrument sheet music / MusicXML / MIDI / TAB、編集・ダウンロード、DAWプラグインを出している。v4の「安い後処理で勝つ」仮説は、Klangioが既に商品面で踏んでいる領域を過小評価している。([klang.io](https://klang.io/transcription-studio/))

- **Songscriptionの法務・データ調達姿勢を軽く見ている。** 公式発表では $5M 調達だけでなく、public domain、アーティスト/音楽事業者との提携、publisher deal を明記している。これは「成果物非関与で逃げる」より、権利処理込みで市場を取りに行く戦略に見える。([songscription.ai](https://www.songscription.ai/blog/seed-funding-announcement))

- **「G0は条件付き通過済み」は言い過ぎ。** G0の本質は「3-6人月で勝てる崩れ方か」だが、その判定はG1結果がないとできない。現時点では通過ではなく、`G0: 未判定。ただしSOTA自社開発案は棄却済み` が正しい。

- **「机上は終了」は早い。** MuseScore NoteVisionの一次資料、Songscriptionの月次成長、Klangioの人数/創業年、$15k MRR上限、スコアメーカーZEROの役割は、v4本文上では投資判断に使われているのに一次資料で封緘されていない。机上調査を増やす必要は薄いが、既存事実の出典固定はまだ必要。

**最終判断**

プランBは続行可。ただし Round 4 でやるべきは「v4追認」ではない。次の3点を直さないなら、G1実施結果は投資委員会資料として採用不可。

1. `mir_eval`に加えて、MusicXML出力対象には`MV2H/MUSTER`または同等の譜面構造評価を入れる。
2. 弁護士3問は、サービス仕様別の事実表付き質問に分解する。
3. G0を「条件付き通過」から「未判定」に戻し、3-6人月仮説を限定ケースのスパイク仮説に格下げする。
