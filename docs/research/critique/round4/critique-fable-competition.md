**観点:** OMR/AMT区別の一貫性、MuseScoreオプション脅威の定点観測の実装度、時間窓18-24ヶ月内の弱点埋め確率、堀T4の実在性、ベンチ対象の最終リスト。

## 総括：v4は Round 3 の指摘を誠実に吸収したが、「観測設計」と「堀の検証」が言葉だけで手順化されていない

v4 §1の AMT/OMR 再定義は正しく、Round 3（grok・competition）の指摘を正面から反映している。カテゴリ誤認は解消された。しかし残る3つの穴が投資判断上は致命的である：**(1) MuseScore 定点観測が「継続タスク化」と書かれるだけで観測手順になっていない、(2) 時間窓18-24ヶ月内に競合がリズム/整譜を埋める確率の明示評価が本文から欠落（§1で「時間窓18-24ヶ月」と数値だけ置き、その根拠となる埋め確率の分解がない）、(3) 堀T4を「唯一の時間耐久性のある堀候補」と昇格させたが、同じ堀の内側にカワイ（スコアメーカー）・ヤマハという国内既存勢が先に座っている問題を検査していない。**

## 検査1：OMR/AMT区別の一貫性 — おおむね一貫。ただし2箇所に残留リスク

v4全体でOMR（画像→譜面）とAMT（音声→譜面）の区別は Round 3 の grok 指摘どおり修正され、以下は正しい：

- §1「MuseScore NoteVision（2025年7月）はOMR」→ 直接競合から除外（正）
- §5 対象外に「MuseScore NoteVision（OMRでありAMTではない）」と明記（正）
- 出典表で grok の「OMR訂正」を根拠として正しく引用（正）

**残留リスク2箇所：**

1. **§4 T4「スコアメーカー文化圏接続」** — スコアメーカー（カワイ）は**OMR＋編集が主力**の製品である（Round 3 grok の実在製品チェック表でも「主にOMR＋編集」と分類）。堀T4を「AMT下書きの手直しレイヤへの接続」と位置づけるなら問題ないが、v4本文は「スコアメーカー文化圏接続」を堀候補と書くだけで、**接続先がOMR製品であること＝AMTの出口として使う設計なのか、OMR資産と競合するのか**が曖昧。ここはOMR/AMTの境界が再び溶けかけている。

2. **§5 参照値「スコアメーカーZERO無料版（国内文化圏の手直しレイヤ）」** — これをベンチ参照値に入れるのは、あくまで「手直しレイヤ」としてであって、**AMT精度の比較対象ではない**（スコアメーカーは音声入力採譜が主機能ではない）。役割が「客観指標のみ軽く取得」と書かれているが、mir_eval（Note F1/Onset F1）はAMT出力にしか定義できない。OMR中心製品にNote F1を測る意味を、封緘前に明文化しないと測定者が混乱する。

**結論：** カテゴリ誤認そのものは解消。ただしT4と§5参照値の2箇所で「スコアメーカー＝OMR」という属性が暗黙化しており、封緘版で役割定義に一文追記すべき。

## 検査2：MuseScore「AMT参入したら詰み」の定点観測 — 手順になっていない（最大の穴）

v4 §1は「AMTロードマップの定点観測（フォーラム・公式発表）を継続タスク化」と書くのみ。これは**観測の意図であって観測の設計ではない**。「継続タスク化」は誰が・何を・どの頻度で・どの閾値で警報を上げるかを一切定義していない。定点観測は具体手順がなければ実行されず、実行されなければ「詰み」の到来を検知できない。以下を封緘版に要求する：

- **何を見るか（観測対象の固定）:**
  1. Muse Group公式ブログ／リリースノート（NoteVisionの次段としてのAudio-to-Score告知）
  2. musescore.org フォーラムの音声採譜要望スレッド（Round 3で既出：node/379835）の公式回答の有無
  3. MuseScore Studio のβ／nightly ビルドの機能フラグ（AMT関連の実験機能）
  4. Muse Group の求人（beat-tracking / audio ML エンジニア募集は着手の先行指標）
- **どの頻度で:** 月1回（NoteVisionが四半期単位で機能拡張している実績から、月次で十分・週次は過剰）
- **どこに記録するか:** ベンチ台帳と同じ場所に「MuseScore-AMT観測ログ」列を持ち、観測日・ステータス（変化なし／要望への公式言及／β実装／正式発表）を1行追記
- **警報閾値（事前固定）:** 上記いずれかで「β実装」以上を確認 → 即エスカレーション（プランB進行中でも堀T4の再評価を強制。無料本体機能に価格前提が飲まれるため）

このレベルまで落とさない限り、v4の定点観測は「気をつける」と同義であり、Round 3 competition批判（唯一の残存デスク調査）への回答として不十分。

## 検査3：時間窓18-24ヶ月内の「弱点埋め」確率評価 — 数値の根拠が本文にない

v4 §1は「時間窓は18-24ヶ月と見積もる」と結論だけを置く。だがRound 3 competition批判の核心は**「全社が崩れることは吉報でなく、全社が次に直す場所を特定したに等しい」**という動学であり、これに答えるには「各競合が18-24ヶ月内にリズム/整譜を埋める確率」を分解して示す必要がある。v4にはこの分解がない。以下が欠けている：

- **Songscription（$5M・月次60%成長）:** 資金使途が公式に「対応楽器・出力フォーマット拡張」でありリズム精度は後回し（Round 3 competition 批判2）。→ **「解けない」でなく「優先順位が後」**。beat-tracking は madmom 等15年の蓄積があり統合問題。18-24ヶ月内の埋め確率は**高**。v4はこれを§1で言及せず、G0の「安く勝てるか」列に丸投げしている。
- **Klangio（独7名・10年）:** 製品幅（8楽器・API・DAWプラグイン・記譜ソフト連携）は先行占有だが、リズム精度への集中投資の兆候は薄い。埋め確率**中**。
- **AnthemScore／Basic Pitch:** 資金・成長速度の点でリズム専任投資の動機が弱い。埋め確率**低〜中**。

**要求：** §1または§5判定表の脇に「埋め確率マトリクス（競合×埋め確率×根拠）」を1枚追加し、18-24ヶ月という窓の下限（18）はSongscriptionが埋めるまでの時間、上限（24）は追随各社を含めた保守側、という**窓の導出過程**を明記すべき。現状は数値が天から降っている。

## 検査4：堀T4（日本語UX・国内配布）は本当に堀か — 国内既存勢が内側にいる問題が未検査

v4 §4 T4は日本語UX・国内配布チャネル・スコアメーカー文化圏接続を「時間耐久性のある唯一の堀候補」に昇格させた。Round 3 competition批判「7条件に言語的堀が1つも含まれていない」への回答としては方向は正しい。**だが堀を昇格させただけで、その堀の内側に誰がいるかを検査していない。** これは Round 3 が Songscription の日本語化リスクだけを見て、国内既存勢を見落とした欠落をそのまま引き継いでいる。

- **カワイ（河合楽器／スコアメーカー）:** 国内記譜ソフトのデファクトで、日本語UX・国内配布チャネル・文化圏の**内側に既に座っている**。堀T4が「日本語UX＋国内配布」なら、個人参入案はこの堀を新規に築くのではなく、**カワイが既に占有する堀に後から入る**ことになる。堀は「外敵（Songscription）を防ぐ」が「先住者（カワイ）には無力」。
- **ヤマハ:** 楽器・音楽ソフト・教育チャネルを持ち、AMTを国内向けに載せる資本と配布力がSongscription以上。ヤマハが日本語AMTを出す確率は、堀T4を無効化する第2のオプション脅威であり、MuseScoreと並べて定点観測すべき対象。
- **含意：** T4は「Songscriptionの日本語化に対する堀」ではあっても「国内既存勢に対する堀」ではない。堀の非対称性（外敵に有効・先住者に無効）を明示せず「唯一の堀候補」と書くのは、Round 3 が Songscription で犯した見落としの反復。**堀T4は「防御対象＝誰から守るか」を明記して初めて堀になる。** 現状は「日本語だから安全」という言語中立性の裏返しの楽観。

## 検査5：ベンチ対象（G1）の最終リスト提案

v4 §5の対象分けは妥当だが、上記検査を踏まえ以下に修正提案する：

| 分類 | 対象 | v4からの変更 | 理由 |
|---|---|---|---|
| 撤退判定用 | Songscription / Klangio / AnthemScore | 維持 | AMT直接競合の主力3社。到達済みなら撤退（正） |
| 参照値（下限） | Basic Pitch（OSS） | 維持 | AMT下限の客観指標 |
| 参照値（手直しレイヤ） | スコアメーカーZERO無料版 | **役割を明文化** | AMT精度比較ではなく「国内手直しレイヤの工数基準線」。mir_evalは付さず手直し比率のみ取得 |
| **定点観測（ベンチ非対象）** | MuseScore／**ヤマハ**／**カワイのAMT参入** | **ヤマハ・カワイを追加** | いずれも堀T4を無効化しうるオプション脅威。ベンチでなく月次観測ログで追う |

**結論：** ベンチ実測対象は v4 の5本（撤退判定3＋参照2）で過不足なし。追加すべきは**測定対象でなく観測対象**——ヤマハ・カワイのAMT参入をMuseScoreと同じ定点観測テーブルに載せること。堀T4を昇格させた以上、その堀を崩す先住者を観測しないのは論理矛盾。
