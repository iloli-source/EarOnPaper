## 1. 代替3仮説（α/β/γ）それぞれの脆弱性

### 1.1 仮説α：日本の権利処理込み B2B SaaS（音楽教室・音大・放送局）

**委員会評価：最も“それっぽく”見え、最も高くつく罠。**

#### 需要は実在するか？
- v2自身が書く需要の実像は「市販譜がないマイナー曲を弾きたいピアノ愛好家」が最頻出。これは**B2Cの個人需要**であり、教室・音大・放送局の購買理由ではない。
- 音楽教室の本業は**市販教材・既刊譜に沿ったレッスン**。採譜SaaSは本業の中核コストセンターではない。
- 音大は「研究ツール」「個人研究費」「科研費」で買うことはあっても、**ARRの立つSaaS契約**になるかは別問題。
- 放送局は権利処理のプロが社内/既存ベンダーにいる。**後発スタートアップが「権利込み」で勝つ領域ではない**。

**未検証：** 「5社ヒアリング」は承認**後**の検証になっている。委員会は**承認前に需要が死んでいるかどうかを見たい**のに、順序が逆。

#### 支払意思はあるか？
- 権利処理込みの価値は「訴訟回避の保険」に近い。保険は**事故確率×損失額**で買われる。  
  音楽教室がAI採譜で摘発される頻度・損失額の定量がない。
- 「権利が面倒だから金を払う」は一見もっともらしいが、実際の購買は  
  (a) 既存の包括契約で足りる / (b) そもそも採譜をしない / (c) 個人に外注する  
  のどれかで**需要が蒸発**しやすい。
- 価格帯の仮説すらない。教室1校あたり月額いくらで、何校で黒字か——**数字がゼロ**。

#### TAMは足りるか？
- 日本の採譜代行を「最大25億円」と自ら切った直後に、その部分集合（法人・権利処理込み）を本命にするのは論理破綻に近い。
- 法人向けは件数×単価は上がっても**件数自体が激減**する。B2BはTAMを大きく見せる修辞であって、ここでは**TAMをさらに縮小するフィルタ**である。
- 放送局・音大は顧客数が桁で少ない。5〜50アカウントで事業になる価格設計が要るが、提示なし。

#### なぜ既存プレイヤーがやっていないのか？
ここがαの最大の赤旗。

| 既存プレイヤー | なぜ「権利込み日本B2B」を本業にしていないか（Occam） |
|---|---|
| Songscription / Klangio | グローバルB2C/APIの方がスケールする。日本の出版社個別交渉は**スケールしないコスト** |
| ヤマハ（Chord Tracker / ぷりんと楽譜） | 既に教材・楽譜・教室エコシステムを持つ。**採譜SaaSを足しても自社譜の販売を食い合う** |
| スコアメーカーZERO / 国内デスクトップ | 買切・業務ソフトで十分。SaaS化する理由が弱い |
| Piascore | 楽譜**販売**の許諾導線。生成SaaSではない |

**解釈：** 空白ではなく、**「権利込みB2B採譜SaaS」は単体事業として旨味が足りない**。  
v2は「グローバルが薄い＝機会」と読んだが、委員会は「グローバルが薄い＝経済合理性がない」と読む。

#### αの追加致命傷
- 補遺が推す「完全ローカル処理」と、αの「B2B SaaS」は**構造的に敵対**する（後述）。
- Arrangement/難易度調整が実需要に直結すると自ら書いておきながら、編曲権でそれが**包括許諾外**だと認める。  
  **需要の中心機能が法務で売りにくい**——これがαの中核矛盾。

### 1.2 仮説β：特定ニッチ（邦楽・sargam・簡譜等の非西洋記譜）

**委員会評価：ロマンはある。TAMとWTPが無い可能性が高い。**

#### 需要は実在するか？
- **簡譜（中国）：** 記譜需要はある。しかしRound 1自身が「AudioJamが月額約¥13相当で扒谱市場を制圧。価格破壊済みで参入妙味薄」と書いている。  
  しかも中国語圏には「来音制谱」「爱扒谱」等の**簡譜/扒谱系ツールが既に存在**する。  
  「西洋ツールが未対応」≠「市場が空いている」。**ローカルプレイヤーが価格破壊している**。
- **sargam（インド）：** 手打ち・共有アプリ（Swarlipi等）と学習アプリ（Riyaz等）はある。AI自動採譜プレイヤー不在は事実かもしれない。  
  しかし「プレイヤー不在」はしばしば**支払意思不在**の結果である。手打ちで足りる、学習は耳と師範で足りる、といった文化的代替がある。
- **邦楽・三線・民謡：** 実在するニッチ需要は認める。だが**愛好家コミュニティ規模・年間外注金額・デジタル化意欲**のどれも未計測。  
  「データセットが不足している」は研究課題としては正しいが、**事業機会の証明にはならない**。

#### 支払意思・TAM
- ニッチ特化は**データ構築コストが固定費で重い**。合成データ＋実録音収集＋専門家ラベリング。
- 顧客は専門家・愛好家・地域文化団体。ARPUは低く、英語圏SaaS価格（$10–30/月）をそのまま載せにくい。
- βが成功しても、出口は「小さな専門ツール」か「大企業への機能売却」。**VC的リターンの物語には乗らない**（それで良いなら、最初から「研究室プロダクト」と書け）。

#### なぜ既存がやらないのか？
1. データが少ない  
2. 評価指標が西洋五線譜前提  
3. **市場が小さい**  
4. 成功してもグローバルARRに効かない  

1–2だけを強調し、3–4を「空白」と美化するのは、v1の「OSS空白＝勝ち筋」の**ニッチ版コピペ**である。  
v2は誤り①で「空白はマネタイズ不能のシグナル」と学んだはずなのに、βで同じ誤りを繰り返している。

#### リズム/拍子「唯一の技術的勝負所」について
- 補遺3は「$5M企業でも未解決＝攻めるなら唯一」と言う。  
- 委員会の読みは逆：**$5Mと7年の先行があっても解けないなら、あなたが解ける確率は低い**。  
- 解けたとしても、Songscription/Klangioが**同じ部品（Beat This! 等）を載せる**までの時間差は短い。技術勝負所は**モートにならない**。

### 1.3 仮説γ：音楽教育向け「AI採譜の精度評価ツール」

**委員会評価：3仮説中いちばん“研究寄り”で、いちばん事業にならない。**

#### 需要・支払意思
- 採譜フリーランス5名インタビューは「課題の有無」は測れるが、**SaaS課金のWTPは測れない**。  
  彼らは構造的に薄利（v2自身の指摘）。ツールに月数千円払う余裕がある層かは疑わしい。
- 音楽大学・MIR研究は**無償OSS・論文指標（MUSTER/MV2H）**で足りることが多い。有料評価SaaSの先例が示されていない。
- 「採譜者の仕事ツール」はSongscriptionが**for-transcribersを既に商品化**している。評価特化で勝つには、  
  **評価そのものが業務必須になる規制・標準・検定**が要る。存在しない。

#### TAM
- 評価ツール市場は本市場の**1〜2桁小さいメタ市場**。  
  本市場（採譜）自体が小さいと認めた上で、その評価ツールを本命にするのは、  
  **沈む船の船長向けコンパスを売る**行為に近い。

#### なぜ既存がやらないのか？
- 評価は論文コミュニティとOSSで進む（v2が挙げる指標そのものが証拠）。
- 商用プレイヤーは**自社品質の開示を避けたい**。第三者評価SaaSは彼らの敵であり、提携もされにくい。
- 要するに**誰も食べられないから空いている**。

## 2. 設計原則の矛盾（ここがv2の知的中核の崩壊点）

v2補遺が推す原則候補：

1. 完全ローカル処理（複製主体リスク回避）  
2. 忠実採譜＝複製の範囲に留める  
3. 学習は合成データ主体  
4. 「音楽学習ソフトウェア」市場に位置づけ  

**これらは同時に満たすと、売れるものが消える。**

### 2.1 ローカル処理 × 成長SaaS

| ローカルの帰結 | SaaS投資家が嫌う理由 |
|---|---|
| サーバーで音源を持たない | 利用ログ・改善ループが弱い |
| 更新がアプリ配布依存 | 機能リリース速度でクラウド勢に負ける |
| 従量課金・シート課金が難しい | 買切 or ライセンス販売＝AnthemScore経路 |
| ネットワーク効果なし | データモートが育ちにくい |
| エンタープライズ管理（SSO、監査、中央制御）が弱い | αのB2Bと相性が悪い |

AudioScore型EULAで30年生きた、は**生存の証拠**であって**成長の証拠ではない**。  
投資委員会は「潰れない」より「伸びる」を買う。ローカル原則は**法務の防御線としては正しいが、成長仮説の自殺線**である。

「ローカルでもサブスク」は可能だが、実態は**古いデスクトップの延命課金**になりやすい。  
Songscription型の「クラウドで改善が回る」モデルとは別カテゴリ。  
**α（B2B SaaS）を語りながらローカル原則を掲げるのは、スライドの左右で別会社の話をしている。**

### 2.2 忠実採譜に留める × 差別化・課金

v2が正しく認めた事実：

- 実需要は**難易度調整・アレンジ込み**が多い  
- Arrangement Modeは編曲（27条）で出版社個別許諾  
- 忠実採譜は複製で保護もされにくい（バンドスコア事件の含意）

つまり：

- **売れる機能**（易しくする・楽器を変える・発表会用に直す）→ 権利地獄  
- **売れる機能を捨てる**（忠実のみ）→ 課金理由が「便利な私的ツール」に縮小  
- 成果物は著作権で守れない → **コピーされやすい**

v1で指摘された「編曲ジレンマ」は**解消されていない**。  
「忠実に留める」はジレンマの片側（安全側）に寄り、**事業として薄い側**を選んだだけである。  
差別化は「少し正確な耳コピ」になり、AnthemScore/Klangio/Songscription/無料モデルと**正面衝突**する。

### 2.3 合成データ主体 × 整譜品質

- Klangioが合成データで学習しているのは**法務回避の先行事例**としては重要。  
- しかしKlangio自体が「beta」「complex recordings unreliable」と評される。  
  **合成データ戦略＝品質の勝利、ではない。**
- 整譜・拍節・声部分離・運指・可読性は、**実演奏の人間的な揺れ・録音条件・ジャンル慣習**に依存する。  
  合成で音高は学べても、**「読める譜面」の美学と慣習は合成しにくい**。
- MAESTRO等NCデータ問題を避けても、**整譜SOTAに必要な教師信号（きれいな最終譜）**は別途高い。  
  合成MIDI→きれいなスコアのペアを量産できるなら、それはそれで**本体事業より難しい研究課題**。

### 2.4 原則を同時採用した場合の製品像

正直に書くとこうなる：

> オフラインで動き、アレンジせず、合成データで学習した、  
> 西洋五線の忠実採譜ツール（場合により邦楽等ニッチ）。  
> 成長市場の学習アプリを標榜するが、レッスン・ゲーミフィケーション・カリキュラムはない。

これは**AnthemScoreの日本語版＋α**に見える。  
投資案件ではなく、**個人開発〜小規模プロダクト**の範囲。

## 3. 「音楽学習ソフトウェア市場に乗る」は実体か、すり替えか

### 3.1 定義のすり替え（クラシックなTAM詐欺）

v2のレトリック：

- Sheet Music：$375M・CAGR ~1% → 乗らない  
- Music Notation Software：$500M→$1.2B・CAGR 10.5%  
- Music Learning Apps：$2.1B→$6.8B・CAGR 13.7% → **こっちに乗る**

**問題：** 市場レポートのラベルを変えただけで、**自社の提供価値は採譜のまま**である。  
Yousician / Simply Piano / Flowkey が成長している理由は：

- レッスン設計  
- 進捗・ゲーミフィケーション  
- コンテンツライブラリ  
- 習慣化・サブスクUX  
- （一部）リアルタイムフィードバック  

**「音源→譜面」ではない。**  
採譜エンジンを学習アプリ市場に帰属させるのは、  
「OCRを作るのでAdobeのTAMを使います」に近い。

### 3.2 数字そのものの脆弱性
- 二次市場調査（Verified Market Reports系など）は**定義がレポートごとに激しくぶれ**、方法論が不透明なことが多い。Round 1自身も「ばらつき大」と書いている。  
- そのばらつきを認めた上で、都合の良いCAGRだけを戦略の柱にするのは**知的に不誠実**。  
- 「$2.1B」は music collaboration 等、**別カテゴリの同額数字と混同しやすい**金額帯でもある（カテゴリ名のインフレに注意）。

### 3.3 実体を持たせる条件（v2は満たしていない）

学習市場に本当に乗るなら、最低でも：

1. 学習者ペルソナと週次利用シーン  
2. カリキュラム / 曲ライブラリ戦略  
3. 採譜が学習ループのどこで必須か（代替手段との比較）  
4. 競合学習アプリが採譜を内製・提携しない理由  
5. 教育価格と解約率の仮説  

v2にあるのは**ラベルの張り替えだけ**。  
「位置づけを変える」はマーケティング文案であり、**事業ピボットの定義ではない**。

## 4. Round 1 調査の残存バイアス・未検証事項

### 4.1 最大の残存欠陥：Fが直っていない

v2本文の表が正直に列挙している：

- 競合実機ベンチ：**未実施**  
- ユニットエコノミクス実数：**未実施**  
- 弁護士の形態別権利分析：**未実施**  
- ユーザーインタビュー：**未実施**  
- 国内採譜需要の年間フロー一次データ：**未実施**

にもかかわらず総合判定は「No-Go → 3検証後に3仮説を再提案」。  
**仮説の数だけ増えて、証拠は増えていない。**  
これは調査の成熟ではなく、**ストーリーの分岐**である。

### 4.2 「反省したデスクリサーチ」バイアス

| バイアス | 残存の仕方 |
|---|---|
| 空白探し中毒 | 整譜空白→日本権利→非西洋記譜→評価ツールへ退避 |
| 成功事例の過大一般化 | Sony MPへの編曲許諾「約4日・無料」個人事例を、事業可能性の空気づくりに使用（本文は一般則も書くが、印象操作としては残る） |
| 競合公式・比較記事依存 | Songscriptionの数字（15万ユーザー、月次60%成長）は**調達PR文脈**が強い。コホート・有料率・解約率なし |
| 失敗談の系統収集不足 | Round 1批判でReddit/YouTube失敗談の未収集を指摘済み。v2でも体系化されていない |
| 研究SOTAの事業転用 | AMT Challenge F1、MIDI2ScoreTransformer等は「難しさの証明」には使えるが、「我々が勝つ」には使えない |

### 4.3 法務補遺の過剰な安心材料化

- 「30条の4で学習は原則適法」は**入力側**の話。出力は複製/翻案が構造的に確定、と自分で書いている。  
  学習適法を強調しすぎると、v1の「私的複製ならOK」と同型の**安心の前倒し**になる。  
- 「完全ローカルなら複製主体リスク回避可能」は**有力な設計指針**だが、判例の銀の弾丸ではない。配布・誘導・事業性が総合考慮される余地を、委員会は残す。  
- バンドスコア事件：成果物が著作物として弱い＝**堀が法的に薄い**。これは脅威。

### 4.4 「3検証」の設計自体が甘い

1. **競合実機ベンチ**  
   - 「手直し30分以内」は良いKPIの芽。  
   - しかし**誰の手直し時間か**（プロ採譜者 vs 中級者）、**合格ラインの事業含意**（30分でも単価が合わない等）が未接続。  
   - ベンチで「競合が既に十分」でも「競合がダメ」でも、**自社が勝つ理由**は出ない。前者はKill、後者は「地獄が確認された」だけ。

2. **弁護士分析**  
   - 必須。だが弁護士は「グレーを白と言わない」。  
   - 形態別マトリクスは**リスクの地図**であり、事業機会の地図ではない。

3. **ユニエコ1枚紙**  
   - 代行・SaaS・B2Bの3形態とあるが、v2は代行を棄却済み。  
   - **棄却済みモデルの計算に時間を使う**のは、まだ未練がある証拠。  
   - α/β/γそれぞれのP&Lに書き直すべき。

## 6. v2の論理構造に対する追加の刺し

### 6.1 「正しい診断 → 間違った計画」を自分で指摘して、またやっている

> 正しい診断から間違った事業計画を導くことは、技術力があるほど危険

この一文は優秀。  
しかし直後に、同じ診断から**α/β/γという別の計画のタネ**を出している。  
自己批判が**次の仮説生成の修辞装置**になっている。

### 6.2 No-Go なのに「承認前検証」がプロダクト前提

承認前3項目は正しい。  
だが「承認後に検討する仮説」表が、**検証が通った世界の製品案**として既に固定されている。  
人間の組織は「表にある仮説」に予算と感情を貼る。  
**Kill判断の文書に、次のピッチの種を植えるな。**

### 6.3 命名クリアランスまでやっている時点で、プロジェクトは生きている

補遺5でOtohiki/Tonebindまで整理している。  
これは調査の完备性というより、**死に体の案件がブランド作業に逃げている兆候**にも読める。  
名前より、同一音源9曲のベンチと教室1校の予算ヒアリングの方が100倍大事。

## 7. v2のまま進めた場合に最も高くつく間違いトップ3

### 1位：ローカル×忠実採譜×学習SaaSの矛盾アーキテクチャを「原則」として固定する
- 法務上もっともらしい設計を先に固めると、**課金・更新・B2B運用・改善ループ**が全部死ぬ。  
- 後からクラウドに戻すと、今度は複製主体論と隣接権が再点火する。  
- **最初の6〜12ヶ月を、売れない安全装置の実装に溶かす**のが最高コストの失敗。

### 2位：α（日本権利込みB2B）を「本命っぽい」という理由で先行させる
- 営業・法務・個別出版社交渉・教室向け要件定義は**重い固定費**。  
- 需要が「個人のマイナー曲」中心なら、法人パイプラインは空振りする。  
- 空振り後にβ/γへ逃げる頃には、**資金とモチベーションが尽きている**。

### 3位：β（非西洋記譜・邦楽等）のデータセット投資を「空白だから」で正当化する
- ラベリング・専門家・評価設計は研究費が燃える。  
- 中国は価格破壊、インドは学習アプリはあっても採譜課金は未証明、邦楽は極小。  
- v1の「OSS空白」と同じ論理で、**より小さい墓場**に掘り進む。

**次点：** 市場レポートのCAGRを信じ、学習アプリ化のフリをした中途半端なUXに金を焼く（レッスン無しの「学習」）。

## 8. それでも生き残る最良の一手

**製品を作るな。物語を増やすな。証拠だけを買え。**

### 最良の一手（委員会が唯一許容しうる前進）

1. **2週間・固定予算の「破壊的検証」だけ承認する**（開発・命名・アーキ・学習データ収集は禁止）  
2. 成果物は次の3つのみ：  
   - **同一音源ベンチ**：Songscription / Klangio / AnthemScore（＋可能なら国内1つ）× 曲セット。採点は「プロが納品可能にするまでの分」と「中級者が弾けるか」の二系統  
   - **顧客5＋法人3の有償ヒアリング**：支払意思は**仮価格を提示して反応を取る**（「欲しい」は無効、「月X円なら買う/買わない」のみ有効）  
   - **弁護士1回分の形態別権利マトリクス**（ローカル忠実 / クラウド忠実 / 代行 / アレンジの4セルで十分）  
3. 判定ルールを**事前に書く**（例）：  
   - 競合が既に「手直し≤15分」かつ価格≤$15/月 → **即Kill**  
   - 法人が権利込みでも月額単価が合わず、個人WTPも$5未満 → **即Kill**  
   - アレンジ無しでは誰も有料更新しない → **忠実原則を捨てるかKill**  
4. α/β/γは**検証結果が出るまで資料から削除**する。名前を与えない。

### それでも残るなら（条件付きの最小生存）

検証後に残る可能性が相対的に高いのは、華々しいSaaSではなく：

- **特定法人向けの受託＋権利処理オペレーション**（プロダクトではなくサービス。スケールは捨てる）  
- または **評価データセット／ベンチ公開による研究・教育ブランド**（事業というより評判資産）  
- あるいは **既存学習アプリ/記譜ソフトへの部品供給**（自前で市場を取りに行かない）

「音楽学習ソフトウェアの高成長市場に乗るスタートアップ」として生きる道は、  
**採譜を核にしたままではほぼ無い。**  
乗るなら学習プロダクトに生まれ変わる必要があり、それは**別会社の創業**である。

## 9. 最終判定（投資委員会）

| 項目 | 判定 |
|---|---|
| v1比の知的改善 | **合格**（事実誤認の訂正は評価する） |
| 投資可能な事業仮説 | **不合格** |
| α | **需要・WTP・TAM・非参入理由、全て未証明。原則矛盾で自滅しやすい** |
| β | **空白の再燃。小さい墓場リスク** |
| γ | **メタ市場。事業にならない** |
| 設計原則 | **防御にはなるが、成長と差別化を殺す** |
| 学習市場リフレーミング | **TAMすり替え。実体なし** |
| 総合 | **Kill（開発予算ゼロ）。破壊的検証バジェットのみ条件付き可** |

**一言で殺すなら：**

> v2は「間違ったGO」を「正しいNo-Go」に直した。そこまでは良い。  
> しかしその直後に、証拠のないα/β/γと矛盾した設計原則と借り物の成長市場で、**またGOの予習を始めている**。  
> 委員会は予習を認めない。**ベンチと財布の口だけ持って戻れ。**
