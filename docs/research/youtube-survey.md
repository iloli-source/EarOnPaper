# AI自動採譜：完全攻略ガイド（動画深掘り）

動画の主張は「魔法のツールを探すより、正しいプロのワークフローを実践せよ」。AIは全自動ではなく「作業の9割を担う優秀なアシスタント」と定義し、**90対10の法則**（AIが解析9割、人間が仕上げ1割）を軸に、4ステップの「マスタリーワークフロー」を提示する。このワークフロー実践でAIの採譜エラーを30〜50%削減できると主張。

## 4ステップ・ワークフロー
1. **準備（Preparation）** — 最終精度の8割はここで決まる。動画で最重要視。
2. **選択（Selection）** — 目的に合ったツールを選ぶ。
3. **生成（Generation）** — AIに楽譜を生成させる。
4. **仕上げ（Refinement）** — 人間が最後の10%を担う。

## 紹介ツール一覧
| ツール名 | タイプ | 用途・特徴 | 推奨シーン |
|---|---|---|---|
| Klangio（クランギオ） | スペシャリスト | ピアノ/ギター専用モデルを持ち、ソロを寸分違わず採譜 | ピアノソロ・ギターソロを高精度採譜したい |
| ScoreCloud（スコアクラウド） | スピードスター | マイク入力で即メモ化、思いつきを素早く形に | 浮かんだメロディを忘れる前にメモ |

## 精度を左右する技術要素（動画が強調）
- **高品質な音源**: 圧縮MP3ではなくWAVなど情報量の多いファイルを使う。
- **クリーンな録音環境**: 反響のない静かな環境で余計な音を入れない。
- **クリップ分割**: 長い曲は短く分けてAIの混乱を防ぐ。
- **ステム分離（Stem Separation）**: ゲームチェンジャー。ボーカル/ピアノ/ドラム等パート別に分離してからAIにかけると認識精度が劇的に向上。

## 評価軸・ツールの選び方
動画はツールを3タイプに分類して選ばせる:
- **スペシャリスト**（特定楽器特化・高精度）
- **スピードスター**（マイク入力・即時性）
- **ヘビーリフター**（多重音源・複雑曲対応）
選定基準は「楽器の種類」「入力方法（ファイル/マイク）」「音源の複雑さ（単音源/多重音源）」。

## 人間が担う「最後の10%」
AIが苦手で人手が必要な領域として明示:
- 小節をまたぐ連桁（タイ/連符）の整理、細かい休符の整理
- 曲の途中で変わるキー、微妙なテンポの揺れ
- スラー・スタッカート・強弱記号など音楽的表現
- 出力は必ず **MusicXML / MIDI** で書き出し、**MuseScore / Sibelius** 等の専門ソフトで仕上げる（AIツール内編集で済ませない）

## 結論・黄金律
「音源はWAV＋ステム分離で準備 → 用途でツール選択（ピアノ/ギター=Klangio、アイデアメモ=ScoreCloud、複雑曲=Music Demixer/Anthem Score）→ MusicXMLで書き出し専門ソフトで仕上げ」。単調なノート入力はAIに任せ、人間は編曲・演奏ニュアンスに集中せよ、が最終メッセージ。

## 機能一覧への示唆
- **入力前処理を機能化**: 音質チェック/ノイズ警告、長尺の自動クリップ分割、ステム分離を採譜前の標準工程として組み込む（精度の8割はここ）。
- **タイプ別モード or ツール連携**: 「楽器特化」「即時マイク入力」「多重音源」の3モードを想定した設計。
- **フォーマット出力**: MusicXML / MIDI エクスポートは必須。MuseScore等への受け渡しを前提に。
- **AI出力=下書き前提のUX**: 生成結果を「下書き」として提示し、連桁/休符/キー・テンポ変化/強弱・スラーなど人手編集を促す仕上げ支援機能を用意。
- **90対10の役割分担**: 全自動を謳わず「AIが解析、人間が音楽的仕上げ」を製品コンセプトに据える。

## 2. デモされていた機能（入力・楽器・編集・出力）

- **AnthemScore**：入力はWAV/MP3。曲全体or部分選択で変換→スペクトログラム表示→ノート編集→**XML/MusicXML経由でMuseScoreに渡して整形・印刷**という2ソフト連携が定番ワークフロー。単旋律（メリーさんのひつじ）は高精度、多声・複雑曲は誤検出が増えると明言。
- **Moises**：入力MP3/WAV。**2mixをボーカル/ドラム/ベース/ギター/その他/オルガンにステム分離**、各パートのソロ・ミュート・音量・パン調整、BPM/キー自動識別、テンポ変更、キー変更、スマートメトロノーム、区間リピート（Aメロ/サビだけ練習）、**コード表示（耳コピ補助）**。PC/iOS/Android同期。無料枠あり。
- **楽譜作成ソフト比較（Dorico/MuseScore/Sibelius/Finale）**：マウス入力・英字キー（C/D/E）でのピッチ入力・数字キーで音価切替・MIDIキーボード入力。**楽譜の読みやすさ・余白/改行など整形品質**がDAWのスコア機能との差別化点として強調。
- **出力形式**：MusicXML（ソフト間連携の要）、MIDI、PDF、音声（MP3エクスポート）。

## 3. 投稿者・視聴者が指摘する不満/限界

- AnthemScore動画自身が「制限がある」と繰り返し警告：**多声・和音・複雑な曲は誤検出**、変換後は必ず手動編集が前提。
- AI変換MIDIは「エグイ譜面（複雑すぎて非現実的な音符）ができる」という声（実際の動画タイトル）。
- DAW内蔵スコア機能は「表示・確認はできるが読みやすい楽譜にはならない」＝**清書品質が不足**。
- Finale販売終了で「今までのファイルの移行」が大きな不安点（MusicXML移行手順の需要）。
- Moises等は便利だが**採譜＝コード提示止まり**が多く、五線譜/タブ譜への完全自動化は未達という位置づけ。

## 4. 人間の採譜講座から見える「プロのワークフロー」（ツールが再現すべき手順）

高再生の講座系（CANACANA、音大卒〜、ずっしー、ゆゆうた等）から抽出される手順：
1. **キー/スケールの特定**（まず調を掴む）
2. **コード進行の把握**（構成音・度数で理解、CAGED等でポジション特定）＝耳コピの中核。ツールも最優先で自動化すべき層
3. **ベース→メロディ→内声**の順で層を分離して聴き取り（音源分離＝Moisesが担う工程）
4. **リズム/拍の割り当て**（1拍目/裏など）
5. **五線譜への清書と整形**（読みやすさ・改行・余白）＝MuseScore/Dorico層
「ステム分離→コード解析→パート別聴音→清書」という4段が繰り返し登場。ツールが各段を橋渡しできると価値が高い。

# YouTube 英語圏 採譜コンテンツ調査（English Sweep）

調査手法: yt-dlp で16クエリ×検索 + 主要レビュー/比較動画5本の自動字幕を抽出。再生数は取得時点。

## 1. 登場ツールの完全リスト

**AI採譜専業（音源→楽譜/TAB）**
- **Klangio**（studio.klang.io）: 英語圏で圧倒的にYouTube露出が多い。自社チャンネルで大量チュートリアル展開。製品群が細分化 — Melody Scanner（リードシート）、Piano2Notes（ピアノ）、Guitar→TAB/Guitar Pro、YouTube URL直接入力、スマホ録音、Edit Mode（音符手修正）、Songbook。MP3/YouTube→MuseScore連携を強く推す。
- **Songscription**（songscription.ai）: 新興。Suno等のAI生成曲→楽譜デモも登場。レビューはまだ小規模チャンネル中心（再生数千台）。
- **AnthemScore**: スタンドアロン老舗。スペクトログラム表示＋MusicXML/MIDI出力。「テンポ無しの音符の羅列になりがち」との指摘あり。
- **ScoreCloud**（Studio/Express/Songwriter）: 弾いた/歌った音を即記譜。自動リードシート。ギターには弱く単純メロディ向きとの評。
- **Songsterr AI**: 従来はTAB DBだったが**新AI**で音源→楽器分離＋TAB自動生成。YouTube URL入力、ドラム/ベース/ギター分離。ギタリストの検証で「怖いほど正確」と高評価。
- **La Touche Musicale**: 「オールインワンAI採譜エコシステム」を標榜。**楽器別のConvert製品**を量産（FluteConvert / OrganConvert / ViolinConvert 等）— ニッチ楽器特化が特徴的。日本での知名度低い。
- **Meloverse**: ごく新興、デモ動画のみ（再生数百）。

**Audio→MIDI（採譜の前段）**
- **Basic Pitch**（Spotify R&D, 無料/OSS）: ブラウザ＆プラグイン。鼻歌もMIDI化。英語圏で定番の無料選択肢。
- **Prism**（audio-to-MIDI）: 新興で高評価集中。「Logic/既存を破壊するレベル」「velocity・timingのニュアンスまで人間的に捕捉」。ARA対応（Logic/Reaper/Studio One/Cakewalk）。ソロ楽器最適。
- **Samplab / Samplab 2**: AI audio→MIDI VST。無料版あり。ポリフォニック対応。
- **Notegrabber 2**（Navie D）: Samplabの対抗として比較される。
- **Moises**（AI Studio/Studio Pro, Fender統合）: ステム分離＋コード検出＋MIDI変換。DAW化が進行。
- **Chord AI**（iOS）: コード検出＋ステム＋MIDI。
- DAW内蔵/その他: Logic Pro audio-to-MIDI、Ableton、FL Studio、Guitar Pro 8のAudio Import、BandLab、Melodyne（スペクトル編集）。

**耳コピ補助（自動採譜ではない）**
- **Transcribe!**（seventhstring）: 減速/ピッチ変更の定番デスクトップ。プロが最も推す。
- **Transcribe+ / Anytune / Amazing Slow Downer**: iOSの減速アプリ。
- TwelveKeys: 小規模紹介あり。

## 2. デモされた機能・UXパターン
- 入力は3系統: **ファイル(MP3/WAV)アップロード / YouTube URL貼付 / その場録音（スマホ・マイク）**。KlangioとSongsterrはYouTube URL対応が強み。
- 変換前に**テンポ・キー・楽器・楽譜形式**を質問するUI（Klangioは質問が多め＝制御が細かい、と好評）。
- 出力形式の選択: リードシート/フルスコア/TAB/Guitar Pro(.gp)/MIDI/MusicXML。**MuseScoreへのエクスポート→手修正**が定番ワークフロー。
- ステム分離を前段に噛ませる設計が主流（lalal.ai等で楽器を抽出→単音源にしてから採譜すると精度UP）。
- 自社内エディタ（Klangio Edit Mode）で音符を直接修正 → ただし多くはPro課金。
- 無料版は**20秒プレビュー制限**（Klangio/Songscription系）。

## 3. レビュアー・視聴者の不満/限界
- **リズムの不正確さ**が最頻の不満。ルバート/明確なビートの無い曲で崩壊（"we're not quite there yet"）。
- **和音の簡略化・欠落**、テンションコード（例: Em7をEmと誤検出）を取りこぼす。
- **ギターの運指/ポジションが非現実的**（弾けない押弦、実際と違うフレット位置に配置）。
- パームミュート・スライド・ビブラート等の**奏法記号が付かない**（Songsterr検証）。
- AnthemScore/ScoreCloudは「テンポ無しの音符の羅列」「単純メロディ専用」。
- 処理時間: Klangioで約10分待ちの例。
- 「AI採譜は上達を妨げる」という**教育者からの原理的批判**（Sharp Eleven Music）— 耳コピの学習効果を奪う。
- 総評は「単線・クリック録音・ソロ楽器なら実用、複雑曲はまだ人間が勝つ。半年〜1年で急伸するだろう」。

## 4. プロ採譜者のワークフロー（動画から）
- 音源→ステム分離→単音源をVST/Basic Pitchで**MIDI化**→MuseScore/Sibeliusで開き**手で整形・TAB付与**。AIは下書き生成、仕上げは人間。
- 耳コピ派: Transcribe!等で減速・ループ、キー/テンポ/コード進行を自問しながら書き取る（Rick Beato, Open Studio, Max Konyi の高再生連載）。

## 5. 再生数分布から見る英語圏の関心
- **最大の関心は「無料でMP3→楽譜」ハウツー**（Musician Startup 82万回が突出）。「無料」訴求が伸びる。
- Klangioが検索を席巻＝英語圏のデファクト認知。SEO/チュートリアル物量で優位。
- **audio→MIDI**（Basic Pitch/Prism）は制作者層で強い関心。Prism系の比較動画が急増中（新興ホット領域）。
- ギタリスト層で「AIはどこまで来たか」検証動画が人気（Songsterr AI, The-Art-of-Guitar 21万/37万回）。
- 耳コピ・音楽理論（Rick Beato 176万回等）は採譜より桁違いに大きい市場＝「採譜ツール」より「耳を鍛える」需要が本流。

## 探したが見つからなかった領域
- **多声部フルオーケストラ/バンド一括採譜**の成功デモは皆無（全て単音源前提）。
- **ドラム自動採譜**専用AIツールは弱く、手作業/Transcribe!/MuseScore記譜が中心（AI採譜の空白地帯）。
- 日本語圏で見る「歌声→ボーカル譜＋歌詞同期」の総合ソリューションは英語動画では手薄。
- 採譜精度の**定量ベンチマーク動画**（F値等の客観比較）は学術講演（Simon Dixon/CIRMMT）以外に一般向けが無い。

# YouTube 中華圏 採譜（扒谱）コンテンツ調査

対象: YouTube（中国本土からはブロックのため台湾・香港・華僑圏の投稿が中心）。8クエリ×20件 + 主要動画のコメント/概要欄を抽出。字幕は主要動画に中国語字幕が無く、コメント欄と概要欄が最も情報密度が高かった。

## 1. 中華圏で言及されるツール（価格・機能）

| ツール | 種別 | 機能・価格の言及 |
|---|---|---|
| **AudioJam（中国系）** | 音源分離+コード/キー/テンポ自動解析 | 吉他手向け「主奏/節奏ギター音軌の分離」が売り。価格「98元/年」とコメント。買い切りではなくサブスク。デバイス間同期が「terrible」との不満 |
| **Songscription AI** | 音源/YouTube URL→五線譜、Piano Roll編集 | 香港系チャンネルが「使った中で最良のAI五線譜生成」と絶賛。**無料版あり**が強調ポイント。ギターは非対応（ピアノのみ）との質問 |
| **小时间音乐/qiupupu.cn** | 扒谱ツール集約サイト | 5ソフト比較シリーズ（#1ピアノ/#2交響楽/#3汎用）の運営者が自作ツールへ誘導。中国系で最も体系的な比較チャンネル |
| **PianoTrans** | ピアノ特化AI | コメントで「全AI中ピアノ扒谱が最良、音符ごとの強弱まで出る」と高評価 |
| **MT3（Google開源）/ YourMT3** | 多楽器AI採譜 | 交響楽に強いが「電子音楽は音軌と構造の参考程度」。音量が一律最大になる欠点の指摘 |
| **安森扒谱 / WIDI Pro** | 半自動採譜 | WIDIは最初期の定番。安森はWIDIの上位版で自動ノート付加が優秀と評価 |
| **Klangio** | ピアノ/ギターTAB特化 | コメントで「ピアノ扒谱ならKlangio一票」。日本語ガイド動画もあり多言語展開 |
| **Melody Scanner** | 歌→楽譜（sheet） | 台湾「沈老師教室」がレビュー。編集可能な点は好評だが精度に酷評集中（後述） |
| その他言及 | — | Moises（音源分離・月額）、Riffstation（無料コード）、Yamaha Chord Tracker（無料コード自動認識）、Capella Audio2Score Pro、中研院 Omnizart（台湾） |

## 2. 中華圏固有の機能要求（简谱/数字譜）

- **简谱（数字譜）対応要求が繰り返し出る。** 比較動画コメント「有没有办法扒成简谱？」、Melody Scannerコメント「簡譜可以嗎」「請問五線譜圖片可以轉簡譜嗎」など複数。
- 「MIDI转简谱」という専用動画が存在（2771再生）＝五線譜→简谱変換の需要が明確。
- 「老人家吹薩克斯風都看簡譜」＝高齢層・アマチュアは五線譜より简谱という文化的背景。
- ギター層は**TAB譜・コード譜**要求が強い（「low Cのギター扒谱AI」「可唔可以生成CHORD及五線」「生成結他用」）。
- 五線譜が読めない層向けに「直接手写」「動的乐谱（JumpingBar）」等の視認性重視ツールも人気。

## 3. 不満・限界の指摘

- **精度が最大の不満**。Melody Scannerに「扒出来的是啥啊，一点可用性都没有」「試了試和原曲完全不一様」が集中。
- **「精度が悪いから買わない」という購買阻害が明確**: 「本來想買，但和原曲不一樣就不敢買啦」「為什麼和原曲不一樣，想買也不敢買了」＝精度が課金の絶対条件。
- MT3系は音符の強弱を再現せず一律最大音量、電子音楽・低音ギターに弱い。
- UI/言語障壁: 比較動画に「没字幕看不懂」、Melody Scannerレビューに「教英文還是教老人用電腦」「像教幼稚園小朋友」＝英語UI・冗長な説明への不満。
- アップロード失敗（「Upload Audio後總是紅色失敗」）等の実用面のバグ。
- 「扒下来的谱子，朗朗都弹不了」＝出力が演奏不能なほど複雑/不正確という皮肉。

## 4. 支払観（買い切り志向）

- サブスクへの抵抗が見える。AudioJam「98元/年」やMoisesの「月費」に対し、無料ツール（Songscription無料版、Riffstation、Yamaha Chord Tracker、免費簡譜軟體）が高く評価・共有される。
- 「免費」を前面に出した動画（廣東話版・普通話版・「開掛後功力增加十年 免費簡譜軟體」）が高再生。
- 明示的な「買い切り希望」の直接文言は今回の抽出範囲では確認できず（ここは追加調査余地）。ただし精度が伴えば「本来買いたかった」という潜在課金意欲は複数確認。

## 探したが見つからなかった領域

- 主要比較動画に**中国語字幕/自動字幕が付いておらず**、字幕からの詳細抽出は不可（概要欄+コメントで代替）。
- 明示的な「買い切り(一次性付費)を望む」直接発言は未検出。サブスク批判と無料志向からの間接推定に留まる。
- bilibiliの具体的言及がYouTube動画経由で見える形では検出されず（運営者はbilibili併用を明記するが内容は不明）。
