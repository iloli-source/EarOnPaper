# X調査報告：AI生成ギターTAB譜の実ユーザー声とベストプラクティス

**調査日:** 2026-07-20  
**対象:** X（旧Twitter）英語・中国語中心（関連で日本語のSongsterr AI実況も補足）  
**方法:** キーワード検索＋セマンティック検索。宣伝投稿・無関係ノイズは除外し、実体験・奏者視点の投稿を優先。

**総括（事実ベース）**  
- **LLM直出しTAB（ChatGPT / Grok / DeepSeek 等）**への不満は英語・中国語とも明確。  
- **音響→譜系ツール（Songsterr AI / Klangio 等）**は「時短になるが要修正」が主調。  
- **「使えないTAB」の具体パターン**（運指非現実・技法記号欠落・小節番号必須など）を体系的に論じた投稿は **X上では薄い**。  
- **中国語圏のTAB特有慣習**の深い議論は **Xではほぼ見つからず**（コミュニティはB站・小红书・贴吧寄りと推定されるが、本調査はXのみ）。

### 5.2 ユーザーが列挙した「最低限リスト」としては **見つからなかった** もの

以下を **必須要素として列挙した投稿** は、今回のX調査では **見つからず**：

- リズム／音価表記（五線併記 or 符尾）  
- 技法記号 h / p / b / s / PM / 等  
- 小節番号  
- テンポ（BPM）・拍子  
- 指番号（1–4）  
- ピック方向（↓↑）

※ 技法（hammer-on / pull-off）そのものの教程投稿は多いが、**「TABの必須要素」議論ではない**。

### 5.3 間接的な示唆（風刺）

The Hard Times（2019, 英語）の見出し共有：  
**「Guitar Tab Mostly Right Except for Tempo, Tuning, Note Choices」**  
→ テンポ・チューニング・音選びが揃って初めて「正しいTAB」という皮肉。AI専用ではないが、コミュニティの基準感を示す。

## 7. 実務に使える「Xから帰納できる」ベストプラクティス（推測なし・投稿根拠のみ）

投稿から **直接言える** 運用だけ：

1. **LLMに曲のTABを一発生成させない**（複数言語で「almost always wrong / 胡扯」）。  
2. **音響採譜（Songsterr AI / Klangio 等）は下書き**として使い、難所は耳コピで直す（「70%」「細かいところは耳コピ」）。  
3. **弦割当・同時押さえ・開放弦**を人間が物理チェックする（unplayable事例あり）。  
4. **把位はTABの核心情報**として保持する（六线↔五线変換で落ちる）。  
5. **チューニング・カポ・弦数**を設定・表記で固定する（誤認識事例あり）。  
6. 変換・修正ルールは **Bass等への単純転用が効かない**ことがある（Codex事例）。
