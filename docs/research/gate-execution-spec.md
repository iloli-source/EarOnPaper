## §0. 封緘手続き（process指摘の実装）

- **封緘者 = ユーザー本人**（代表）。AIは封緘案の提示のみ。封緘時に「日付＋バージョン＋変更不可宣言」を本ファイル末尾に追記する
- **封緘タイミング:** G2前半・G1着手前の1点。着手後の封緘は無効
- **二段階封緘:** (1) 本仕様の手続き・判定規則を先に封緘 → (2) G1帯境界の数値はパイロット校正（§3.4）後に確定値を追記し封緘
- **改訂ルール:** 測定「手続き」の改訂は ①理由 ②日時 ③判定を有利化していないことの自己申告 を本ファイルに追記のうえ可。**判定「閾値」の実測開始後の改訂は原則禁止**。基準を緩めた事実はログに残し、Round 5 で第三者レビューの監視対象とする
- **本プランの性格（冒頭宣言・grok要求）:** 本プランは教育資産とオプション価値の購入である。4ゲート全通過以外のあらゆる結果は撤退であり、スパイク承認の誘惑に負けない

### 3.3 測定プロトコル（tech-bench封緘条件Cの実装）

- **主指標（改訂2 2026-07-19）:** ①AIの耳ハーネスの複合スコア（音高一致度・出だし一致・テンポ整合・譜面健全性） ②非音楽家聴感の3段階一致。帯境界の実測後封緘の原則（§3.4）は維持し、対象指標をハーネススコアに読み替える
- **参考データに降格（改訂2）:** ~~基準線: 同一整譜者がゼロから耳コピ起譜する時間・手直し比率=（AI出力の手直し時間）÷（フル耳コピ時間）~~ — 音楽家依存のため主枠組みから除外。演奏者向け出力オプションのQA時にのみ採取
- **手直しの操作的定義（演奏者向け出力QA専用に降格・定義自体は保持）:** 開始 =「AI出力をエディタで開いた時刻」。終了 =「①mir_eval Note F1 が基準線起譜と同水準 ∧ ②拍子・小節線に構造誤りゼロ ∧ ③測定者が『この譜面を人に渡せる』と宣言」を満たした時刻。自己計時・タイマーはツール切替時に停止しない
- **手直しツール:** MuseScore Studio に統一固定（全測定・全測定者共通）
- **ブラインド化:** 出力ファイルは測定前にスクリプトで無名ID（T01…）にリネームし、対応表は測定完了まで別ファイルに分離。測定順はランダム化。1人運用の限界（Webツールを自分で叩けば製品が分かる）は「出力取得日と測定日を分離し、測定時は無名ファイルのみ扱う」ことで緩和し、完全ブラインドでないことを台帳に明記
- **測定者・評価枠組み（改訂履歴を保持）:**
  - ~~初版: 熟練1名（代表）を基本~~（撤回: ユーザーは音楽経験なし・楽譜が読めず音感もないと本人申告 2026-07-19）
  - ~~【手続き改訂1 2026-07-19】音楽関係者のつて（謝礼付きレビュアー）に委譲する~~（同日撤回。理由: 下記改訂2のとおり評価軸自体がターゲット非整合と判明したため。履歴として保持）
  - **【手続き改訂2 2026-07-19・現行】主判定を人間測定者から二段判定へ転換する。** ①**第一判定=事前封緘した客観指標**: AIの耳ハーネス（`tools/ai-ears/`）による音高一致度（クロマDTW類似度）・音の出だし一致・テンポ整合・譜面健全性の複合スコア。閾値はハーネスの自己検証完了後・実測開始前に封緘（§0の二段階封緘と同じ規律） ②**第二判定=ターゲットユーザー聴感**: 非音楽家（代表本人＝ターゲットユーザーとして最適な判定者）が元音源と採譜結果の再生音を聴き比べ、「①同じ曲に聞こえる/②部分的に同じ/③別物」の事前固定3段階で判定（楽譜を読む必要なし）。**改訂理由:** 製品は絶対音感エミュレータ（非音楽家ターゲット）であり、「音楽家が直す苦痛」を主基準にすると製品が音楽家の基準に引っ張られターゲットと乖離する（ユーザー指摘 2026-07-19）。**有利化していないことの申告:** 基準を主観に緩めるのではなく、続行バイアス対策を「第三者の耳」から「基準の実測前封緘」に置換するもの。閾値封緘前に実測値を見ることは禁止。音楽家評価（手直し苦痛・演奏可能性）は廃止ではなく**演奏者向け出力オプション（TAB等）のQA専用に降格**。音楽関係者のつては、プロダクトが五線譜PDF出力を実現した後の製品レビューに温存し、G0'/G1では使用しない（`g0-reviewer-brief.md`は同用途に温存）
- **誤り種別分解:** 音高 / 音価 / 拍子・小節線 / 声部分離 / その他ごとに修正所要時間を記録

### 4.2 質問リスト（The Mom Test準拠・そのまま読み上げ）

問1-9はアイデアを一切説明せず過去だけを聞く。アイデア説明は問10の直前まで封印。

1. 直近で「この曲の譜面が欲しい」と思ったのはいつ、どの曲ですか。そのとき結局どうしましたか
2. その譜面を手に入れるのに、時間はどれくらいかかりましたか。何が一番面倒でしたか
3. これまでに採譜や耳コピにお金を払ったことはありますか。誰に、いくら、何回
4. 外注したとき、上がってきた譜面はそのまま使えましたか。手直しはどれくらい必要でしたか
5. 耳コピや外注をやめて別の手段に切り替えたことはありますか。なぜ切り替え（なかっ）たのですか
6. 今、採譜に使っているアプリやツールはありますか。何が不満で、なぜ使い続けて（やめて）いますか
7. もし譜面が5分で8割の精度で出てくるとしたら、残り2割の手直しは自分でやりますか、それとも要らないと感じますか
8. 最後にお金を払ったソフトやアプリは何で、いくらでしたか。買い切りとサブスクどちらでしたか
9. 周りに同じように採譜で困っている人はいますか。その人たちはどうしていますか
9-B1. （B系統のみ）演奏経験がなくても「この音、何の音だろう」「この鼻歌をメロディとして残したい」と思ったことはありますか。具体的にいつ、何の音でしたか
