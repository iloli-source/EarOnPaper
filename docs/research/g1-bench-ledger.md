## §4. 帯境界の二次封緘手順（G1実施時に実行・spec §3.4準拠）

1. **アンカー取得**: 基準線曲2-3本（U1×2・U2×1）を Basic Pitch（TFLite経路。ONNXは使用禁止=台帳§7.1）に通し、AIの耳の複合スコア＋聴感の実測値を取得（OSS無料下限のアンカー）
2. **境界決定**: アンカー値を基準に3帯境界（撤退≥ / グレー / 勝負所≤）をハーネススコア基準で決定
3. **封緘**: 確定値を gate-execution-spec.md 末尾の封緘記録（二次封緘欄）に追記し、ユーザーが封緘宣言。**以後の改訂は原則禁止**（緩めた場合はログ+Round5監視対象）
4. **分散確認**: パイロット1曲でハーネス再現性・聴感安定性を確認してから本測定へ
5. **注意（rhythm-autopsy §5.1の知見）**: 出だし指標は量子化出力に対して基準系で逆方向を示しうる。帯境界の決定時は「同一測定系での相対比較」原則を厳守し、楽譜レベルKPI再定義（Issue #33）の結果が出ていればそれを優先
