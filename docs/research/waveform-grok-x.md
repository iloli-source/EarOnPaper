## 1. 波形表示UIの実ユーザー評価

### 1-1. 「見やすい／美しい／気持ちいい」と評される具体

| 要旨 | 言語・時期 | 根拠の方向 |
|------|------------|------------|
| **シャープな波形レンダリング**が好意的に言及される（Audacity 4.0 alpha: “I love the sharp waveform display”） | 英・2026/1 | 解像度・エッジのキレが「良さ」の言語化ポイント |
| **話者ごとに色分けした波形**（ElevenLabs UI + transcript diarization を改造）が「Beauty」と評価・高保存 | 英・2025/10 | 色＝意味ラベル。単色波形より情報密度が上がる |
| **周波数カラー波形**への強い要望（Abletonに「Serato/Rekordbox風」を何度も要求）。日本語でも AURORA プラグイン紹介 | 英 2017／日 2025–2026 | DJ文脈の「色で帯域が読める」がDAWにも欲しい |
| **効率的LOD（Level of Detail）波形描画**で長尺でも即表示・高FPS → 開発者コミュニティで「最効率」と話題 | 英・2023/12 | 体感＝**ズームしてもカクつかない**こと自体がUX品質 |
| **クリップ単位コンテナ**（Audacity試作: 波形を直接編集しつつクリップ移動も可能） | 英・2021 | 「波形編集」と「アレンジ」の両立が小さなが効く改善 |
| 中国語: **波形更清晰・时间轴更准・更丝滑**（AudioCut） | 中・2026/4 | 数少ない中文の直接評価。**清晰＋丝滑**が品質語彙 |

### 1-2. 「見づらい／使えない」不満の具体

| 要旨 | 言語・時期 | 示唆 |
|------|------------|------|
| 波形移動中に**波形が見えない**／トラック並べ替え不可／ズーム・スクロールがメニュー操作に食われる | 英・2023（高エンゲージ） | **ドラッグ中も波形プレビュー必須** |
| **抽象化されすぎた波形**はナビに使えない（「play bar と変わらない」）。本物の波形なら目視ナビできる | 英・2021 | 装飾用ミニ波形と**作業用波形**は別物 |
| 視覚化が**変な潰れ／伸び**をする（visualizer hate） | 英・2026 | 正規化・ピーク表示の誤実装が即座に嫌悪 |
| Audacityの**線が太くて見づらい** → SoundEngine 等へ逃げ | 日・2024 | 線幅・アンチエイリアスは好みが分かれるが「太い＝精度感↓」層あり |
| **画面が小さく波形が見にくい**（機材・ソフト世代差） | 日・2026 | モバイル／小画面では波形高さ・コントラストが生死 |
| 再生ヘッドが追従しない・波形が消える → カット不能 | 日・2025 | **再生ヘッド同期と波形常時描画**は編集の前提 |
| Cubase UI「めっちゃ見辛い」／Logicは録音向きだがバグ話 | 日・2024 | 全体UI密度と波形可読性はセットで語られる |
| ズームUIが旧態依然で「何もできない」 | 英・2026 | ズーム＝操作入口。ズームが悪いと機能が死ぬ |

### アプリ仕様への示唆（テーマ1）

1. **作業用波形**と**SNS用ビジュアル波形**を分離設計する  
2. デフォルトは**高コントラスト・シャープ**、設定で線幅／塗り／グローを調整  
3. **話者／音源レイヤ／ピッチ候補**ごとに色分け（diarization 的メタデータ連携）  
4. ズーム・スクロール・再生ヘッドは **60fps級・LOD描画・操作中もプレビュー**  
5. 周波数カラーは「上級者オプション」だが、耳コピ層の**欲望が明示的**

## 2. 波形とスペクトログラムの使い分け

### 2-1. 実務言説のマップ

| 表現 | 用途の言説 | 言語・時期 |
|------|------------|------------|
| **波形** | タイミング、アタック、沈黙、カット位置、クリップ、ゼロクロス、ラウドネスの「山」 | 日英とも実務Tips中心 |
| **スペクトログラム** | ノイズ種の見分け（鼻鳴り・リップノイズ）、周波数成分の手動消去、編集痕跡、ピッチ帯の視認 | 日 2022–2024 特に強い |
| **波形＋スペクトログラム重ね** | 耳コピ特化（ループ中に拡大、平均律量子化軸でピッチ強度） | 日・2025（Genkhord/Nplay） |
| **スペクトログラム重ね掛け** | 「耳コピ勢は相当嬉しい」と開発者が明言 | 日・2026 |
| **メルスペクトログラム** | UTAU原音設定で「通常スペクトロより見やすい説」／AI側ではASR・採譜モデル入力として標準 | 日 2026／英 2026 研究・製品 |
| **波形 vs スペクトルの限界** | スペクトロは時間×周波数の妥協表示（不確定性）。「真実ではなく妥協の可視化」 | 英・2026 |

### 2-2. iZotope RX への評価

| 評価軸 | 要旨 | 言語・時期 |
|--------|------|------------|
| 肯定（主流） | スペクトロを見ながらノイズ除去／リップ・ハム・クリックを手動で消せる。**ゼロクロスにスナップ**して切れるのでAudacity不要論 | 日 2022–2026 |
| 肯定（楽しさ） | スペクトロを見るのが「楽しくて興味深い」——子音の形・個性の可視化 | 日 2024 |
| 手順共有 | ボーカル／ナレーション向け簡易ノイズ除去フローが何度も再掲・高保存 | 日 2024 |
| 否定／代替 | **重い・高い**。rnnoise 等 OSS が品質・資源で勝るケースあり | 英 2026・高保存 |
| 限界 | 自動だけでは取り切れず手動必須。クリップ復元は「形は戻るが音は割れたまま」もあり | 日 2022, 2026 |

### アプリ仕様への示唆（テーマ2）

| 層 | 推奨ビュー | 理由 |
|----|------------|------|
| **耳（採譜・音程選択）** | スペクトロ or メル＋平均律グリッド | ピッチは時間振幅では読めない |
| **録音品質診断 F-002** | 波形（ピーク・クリップ・沈黙）＋簡易スペクトロ | クリップは波形、持続ノイズはスペクトロ |
| **音成分の選択抽出** | RX風の**領域選択（時間×周波数）** | 日常雑音から「音程成分」を塗って残すUIと相性 |
| **二層UI** | 上: 波形（いつ鳴ったか）／下: スペクトロ（何が鳴ったか） | 実務者がすでに脳内でこう使い分けている |

メルは「人間の聴覚スケールに近い」一方、**音楽採譜の絶対音高表示**には平均律Hzグリッド付きリニアスペクトロの方が説明しやすい。メルは内部モデル用、UIはHz＋ノート名が無難。

## 3. 録音品質と波形の関係（F-002 音質診断ネタ）

### 3-1. クリッピング

| Tips | 根拠言説 | 言語・時期 |
|------|----------|------------|
| クリップ波形は**頭が平ら／矩形化**。歪みが焼き付き、後で音量を下げても消えない | メーター赤・波形比較の教育ポスト多数 | 日英 2019–2026 |
| 目安: 大声でも **-12〜-6 dB** 付近。0 dBFS 到達＝波形クリップ | OBS／配信音響の実務 | 日 2026 |
| **波形が小さいのに潰れている** → 入力経路・前段の問題を疑う | Cubase 相談 | 日 2026 |
| ストリーミング再エンコードで true peak が持ち上がり端末でクリップ → **1 dB headroom** | 英 2026 | |
| クリップ検出ツール需要: 赤ハイライト＋タイムスタンプ報告 | 日 2026（Audio Clip Checker） | UIとして「赤帯」は既にユーザー言語 |

### 3-2. 低SNR・ノイズ

| Tips | 根拠 | 示唆 |
|------|------|------|
| エンジニアは「波形をじっと見てノイズを探す」 | 英 2024（15年経験談） | 診断画面に**ズーム強制導線** |
| リップノイズ等は**波形とスペクトロ両方に出る** | 日 2022 | 単一ビューでは取りこぼし |
| 鼻鳴りはスペクトロで「独特の形」 | 日 2024 | パターン分類UI（アイコン化）が効く |
| 品質ダッシュボード: loudness / noise / DR / clipping / silence / waveform / spectrogram | 英 2026 HF Space | **F-002の指標セットの雛形** |

### 3-3. 位相

| Tips | 根拠 | 示唆 |
|------|------|------|
| 逆相でモノラル化すると消える＝**位相キャンセル**。複マイクレコ・ステレオ素材で頻発 | 日 2026（スクール公式） | 日常スマホ録音は単一マイクが多く**検出頻度は低め**だが、ステレオ／複数端末同期時に有効 |
| 反射 vs 別イベントは、**立ち上がり時間・エネルギー・スペクトル形**で波形から切り分け可能 | 英 2026（フォレンジック寄りの長文） | 高度診断向け。一般UIでは「L/R相関」メーター程度で十分 |

### F-002 仕様ネタ（波形から見抜くチェックリスト）

```
[ ] ピークが 0 dBFS 近傍で平ら → デジタルクリップ（致命）
[ ] 全体が極端に小さい山 → 低レベル／後ゲインでノイズ増幅リスク
[ ] 波形は小さいのに歪み感／潰れた形 → アナログ段 or 経路クリップ
[ ] 無音部にも細かい抖动 → 低SNR／環境ノイズ
[ ] 沈黙が長すぎる／詰まりすぎ → 編集・切断・自動ゲインの痕跡
[ ] ステレオで L/R が鏡像的・モノで消える → 位相反転疑い
[ ] スペクトロで水平の帯 → ハム・ファン・エアコン
[ ] スペクトロで垂直の縦筋 → クリック・ポップ・編集継ぎ目
```

### 4-2. 中国語圏

- **音声波形のSNS映え事例はほぼ未検出**（「波形」語が金融に奪われている）
- 代わりに、独立ツールが「清晰・丝滑」でUI自体を売りにしている程度

### 日常音採譜アプリへの示唆

| シェア資産 | なぜ効くか |
|------------|------------|
| **Before: 生波形（雑音だらけ）→ After: 抽出後のクリーン波形／ノート列** | 「魔法」が視覚で一瞬伝わる（RXの「rescue」物語と同型） |

## 5. 波形編集の操作感（選択・カット・ループ）

### 5-1. 「気持ちよさ」言説

| 要旨 | 言語・時期 |
|------|------------|
| ループ地点が**波形でピタリ嵌ると気持ちいい**／「波形との闘い」 | 日 2023–2026 |
| **ゼロクロスで切る**とクリックが出ない——超ズームした波形上の「赤線」教育が刺さる | 英 2026 |
| RXのゼロクロス吸着で「切りやすい」 | 日 2025 |
| 無限ループで流しながら**波形をザクザク触る**方が没頭できストレスが少ない | 日 2022 |
| 構成が明瞭な曲は**波形から構成を逆特定**できる；ループ点が決まると「すごい気持ちいい」 | 日 2020 |
| クリップ単位で触れること自体が「tiny but so useful」 | 英 2021 |
| レコボ: 波形上にボーカル解析が出るとサビ頭へ即移動 | 日 2021 |

### 5-2. 「気持ち悪い／ストレス」

| 要旨 | 言語・時期 |
|------|------------|
| 波形ズームが重い・スムーズでない（モバイル版差） | 日 2023 |
| 選択・カットが単語を削る／無音を残す（自動編集の誤差を ms で評価したい） | 英 2026 |
| Descript: 波形上の切れ目精度への不満はあるがコア編集はそこでやる | 英 2026 |
| 抽象波形ではナビ不能 → 操作の前提が崩れる | 英 2021 |

### アプリ仕様への示唆（テーマ5）

1. **ゼロクロス・スナップ**はデフォルトON（OFFも可）  
2. 選択範囲は**半透明ハイライト＋両端ハンドル＋波形ピークに吸着**  
3. ループは波形上に **IN/OUT フラグ＋嵌り度スコア（クリック予測）**  
4. 再生中編集（触りながら聞く）を優先——停止→精密の二段  
5. 自動カット結果は**波形上に候補を重ね、ユーザーが1スワイプで修正**できること

## プロダクトへの統合示唆（絶対音感エミュレータ向け）

```
[録音] 波形ライブ表示 + クリップ赤帯 + SNR簡易ゲージ
   ↓
[診断 F-002] 波形（クリップ/レベル/沈黙）∥ スペクトロ（ハム/宽带噪声）
   ↓
[耳＝選択] スペクトロ上で音程成分を塗り選択（RX的）／平均律グリッド
   ↓
[楽器非依存層] 抽出後のクリーン波形 + ピアノロール/五線
   ↓
[シェア] Before雑音波形 → After抽出 + 楽譜の audiogram（15–30s）
```

**差別化の芯**: DAWは「波形で切る」、RXは「スペクトロで消す」、本アプリは「スペクトロで**音程を選ぶ**」——日常雑音の中から「聞こえた音」をユーザーが指で拾う二層UI。X上の実務知はこの方向を既に支持している。
