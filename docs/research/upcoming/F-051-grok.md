## 1. 成功例

### 1-1. アーティスト公式配布のフォーマットとして GPX が使われる

- **主旨:** 公式TABストアで全リリースのギター/ベースを **GPX + PDF** のデジタルダウンロードとして提供。
- **アカウント:** `@polyphia`（アーティスト／バンド公式）
- **出典:** 2017-08-11 — “Digital downloads … available in GPX and PDF.”

### 1-2. 作曲 → MIDIエクスポート → DAW という実務パイプラインが成立

複数の実務ミュージシャンが、**Guitar Proで打ち込み → MIDIエクスポート → Cubase / EZDrummer 等**を常用している。

## 3. ベストプラクティス（投稿から抽出できる実務知）

> ※「こうすべき」という規範ではなく、**複数投稿で繰り返し観測された成功パターン**。

### 3-1. 直接バイナリ生成が難しいときは **MIDIを中間形式にする**

- PyGuitarProで新規GP作成を諦め、**MIDI経由でインポート**して成立させた事例（研究・開発者）
- 中国語ユーザーも **GP作成 → MIDI → 微調整** を明示

### 3-2. **役割分担:** 記譜はGP、音色・ドラムはDAW

- ドラム：GP打ち込み → MIDI → Groove Agent / EZDrummer  
- 目的を「ラフデモ」「プロジェクトセットアップ」に限定する声が複数

### 3-3. 公式配布は **GPX + PDF の二重提供**

- バンド公式ストアの実績フォーマット（Polyphia）

### 3-4. ライセンスなしでの閲覧は **TuxGuitar等FOSSでGPファイルを開く**

- Ultimate Guitar等のGPタブを読む用途でTuxGuitar利用の声（仏語ユーザー `@danivempire666`）
- FOSS推進側もimport/exportを明示的に訴求

### 3-5. AI採譜は「完成譜」ではなく **たたき台 → 耳コピ補正**

- 時短に有用だが精度・表裏ミスがある、という利用実態（§2-C）

### 3-6. 開発マイルストーンを「valid GP」と「音楽的正確さ」に分離

- ファイルとして開ける／valid であることと、ノート精度は別問題として進捗管理する事例（開発者）

## プロダクト設計への示唆（投稿根拠つき・過剰推論なし）

投稿群から**直接言えることのみ**:

1. **「validな.gp/.gp5を吐ける」こと自体が価値**（開発者の進捗指標）
2. 一方でユーザーが怒っているのは valid 以前に、**歌詞・拍子変更・AI精度・表裏・メタデータ欠落**などの**意味論・編集意味**
3. ライブラリ利用者は **書き込みAPIのDXが悪いとMIDI迂回**する
4. 市場の交換フォーマットとしては **GPX/GP系が依然デファクト**（公式バンド配布、Songsterr/UGエコシステム）
5. 並行して **FOSS互換・DAW直TAB・AI→GP** が三条線で進んでいる

**まとめ一文:**  
X上では「Guitar Proエクスポート」は**交換フォーマットとしての成功（MIDI/DAW・公式GPX・AI→GP）**と、**ライブラリ生成のDX不全・相互運用メタデータ欠落・AI精度・編集破壊的挙動・囲い込み課金**への不満が同居しており、PyGuitarPro名指しの議論は稀だが、ある事例は明確に **「新規作成がawful → MIDI迂回」** である。
