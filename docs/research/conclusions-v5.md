## 1. 前提の更新: MuseScore は既に AMT に手を出している

- **事実:** MuseScore Studio 4.7.4（2026-07-07）のリリースノートに「Import audio to score: File メニューから online MP3→MSCZ converter（ベータ）へのショートカット」が明記。フォーラムではエンジン候補として Basic Pitch / Omnizart が議論されている（grok・一次資料 musescore.org/en/4.7.4）
- **v4の誤りの性質:** 「NoteVision=OMR」の区別自体は正しかった。誤りは「AMT参入は未来のオプション」とした点。**参入は現在進行形**
- **帰結:**
  1. 「参入したら詰み」の条件文は「**参入の品質を今すぐ実測して詰みか否かを判定する**」に置換 → G0'ゲート新設（半日）
  2. 時間窓の起点を2026-07-07にリセット。保守的見積もりは**残存半年**
  3. 競合集合を拡張: OSS下限に MuScriptor（2026-07公開・多楽器）を追加、用途別支配者として Songsterr AI（TAB）/ Chordify（コード）/ Moises（前処理）、価格破壊として AudioJam 等中国勢を観測リストに明記
  4. 定点観測は「月次・フォーラム確認」から「**週次15分・自前でβを叩く**」に格上げ、kill switch 3条件を事前封緘（spec §7）

### 部分反映・却下（理由つき）

- **TAB/コードのKPI分離（grok）:** 部分反映。本プロジェクトの主戦場は譜面系（U1/U2）に固定し、U3ギターTABは**Songsterr支配によりスコープ外の判定材料**としてベンチに1曲だけ含める。TAB市場への参入判断は本ループの対象外
- **インド市場の再調査（grok P2）:** 却下（低優先ウォッチのまま）。「調査したので不在」の出典はRound 2レポートに固定済み
- **測定者2名の完全な独立測定（codex）:** 部分反映。1人運用の現実制約から、準初級測定者の外部調達は「デモ到達したG2インタビュー対象者への謝礼付き依頼」で代替可能とし、不可能な場合は「熟練1名＋ファイル無名化＋測定順ランダム化」に縮退（限界を台帳に明記）

## 3. 4ゲートの最終定義（詳細は spec）

**実施順序はコスト昇順・撤退シグナル最速順。どのゲートでも「進める積極的理由が封緘基準に照らして立証できない場合のデフォルトは撤退」。**

| ゲート | 内容 | 判定 | コスト |
|---|---|---|---|
| **G0'** 実機初手 | MuseScore audio2score β＋MuScriptor に固定3曲（U1/U2/U4）を通し手直し感を確認 | 準初級相当の手直しが軽い（比率≤40%目安）→ **残ゲート実施せずクローズ** | 半日・現金0 |
| **G2前半** 需要 | 発注経験者中心にインタビュー問1-9（質問文は着手前に封緘） | 支払実績の実在を確認。デモ到達10人を目標 | 3-5日・謝礼1-2万円 |
| **G1-A** 技術ベンチ | 用途マトリクスn=9・基準線先行測定・帯校正→封緘→測定 | 3帯判定（spec §3）。明確撤退帯→即終了 | 3日（+7暦日の絶対期限）・現金1-2万円 |
| **G2後半** 需要 | デモ（G1-AのOSS出力転用）提示＋予約金¥3,000決済オファー | 決済0件→撤退 / 1-2件→追加5人 / 3件以上→通過 | 1-2日 |
| **G0** 経済性確定 | G1の誤り種別分解から「埋め合わせ工数≤3人月（逸失利益≤405万円）∧現金≤10万円」を判定 | 超過→撤退（勝負所濃厚帯でも） | 机上 |
| **G3** 法務 | 弁護士4問＋事実別表。二段発注（口頭相談3-10万円先行） | 1問でも危険→全体撤退。3値合成規則は spec §5 | 5-10営業日・15-25万円 |
| **スパイク** | OSS出力＋後処理プロト。**1人月構成を優先**（3人月は上限であって計画値でない）。8暦週の絶対期限 | 手直し比率20pt改善未達→撤退 | 最大3人月=逸失利益240-405万円 |

**総コスト（全通過の最大ケース）:** 時間約3.7人月・現金22-39万円＋法務別枠・機会費用332-497万円。**9割超が最終スパイクに集中**するため、実質の投資判断は「スパイクにGOを出すか」の一点であり、G0'〜G3はその前さばきである（economics）。

## 4. 最終判定の枠組み

- **Go/クローズの判定はユーザー本人が下す。** Round 5 のAIレビュアー群は実測結果への批判（バイアス・測定不備の洗い出し）と判定材料の提示のみを行い、**投票権を持たない**（AI合議は相関の高いレビュアーによる非独立試行のため）
- 順序: ①全ゲート結果が出揃う → ②Round 5 でAI群が実測結果を批判 → ③ユーザーが最終判定文書に署名
- 判定会の必須入力: 4ゲート結果表＋封緘版判定表との照合記録。口頭・印象での判定は不可
- **全ゲート非撤退でも自動Goにしない**: 必要条件クリアに過ぎず、「進める積極的理由」の立証がなければデフォルト撤退

## 5. タイムライン（process試算）

- **早期撤退（G0'またはG1-Aで撤退帯）:** 封緘含め**約6-8営業日**でクローズ（最頻・最安）
- **グレー進行→法務まで:** 約20-28営業日（3-5.5週）
- **全通過→スパイク:** 上記＋最大60営業日（**約4ヶ月**。v4自認のとおり確率の低い経路）
- 律速: 封緘完了とG3弁護士の先方リードタイム

## 6. 経営判断の記録

- **2026-07-18 ユーザーがプランB（4ゲート逐次検証）を採択。プランAは却下。** ただし redteam の移植提案（絶対期限・グレー=撤退・需要先行・外部即停止トリガー）を spec に組み込むことで、プランBは「最速撤退寄りの4ゲート」として運用する
- redteam の残存論拠「経営文脈の機会費用（資金繰り危機・SES/取引先A社の現金化機会）」は、即時全停止シグナル④⑤（spec §7）として休眠トリガー化。**発火時はプランBをプランA相当（即クローズ＋台帳保存）に自動降格**
- 本プランの性格の明文化（grok要求）: **これは教育資産とオプション価値の購入である。4ゲート全通過以外のあらゆる結果は撤退であり、スパイク承認の誘惑に負けない**

## 7. 総合判定

| 項目 | v4 | v5 |
|---|---|---|
| 総合判定 | No-Go継続・4ゲート | **No-Go継続・実行仕様確定**。G0'の半日実測から実行フェーズ開始 |
| 前提の鮮度 | MuseScore=未来のオプション脅威 | **MuseScore AMTベータ出荷済み**を織り込み。時間窓残存は保守的に半年 |
| 封緘状態 | 未封緘（Round 4レビュー待ち） | **spec封緘可**（帯数値のみパイロット校正後に確定・封緘の二段階） |
| 次アクション | Round 4 | **G0'実施 →（生存時）G2前半 → G1-A**。Round 5 は実測結果への最終批判とユーザー最終判定の材料提供 |

**v5の結論を一行で:** 検証装置の設計は完成した。次の半日（MuseScoreβ＋MuScriptorに3曲を通す）が最初の実測であり、そこで手直しが軽ければ残りのゲートを実施せずクローズする——プランBを儀式にしない唯一の方法である。
