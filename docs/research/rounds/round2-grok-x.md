| パターン | 例 | 読み取り |

## 3. 買い切りデスクトップ採譜ソフトのユーザー動向

### 3.1 AnthemScore

| 種別 | 内容 | 時期 |
|------|------|------|
| 購入直後バグ | 「Purchased today… exported file is empty」 | 2025-09 |
| 仕様誤解 | 購入後に **2分制限** に気づき代替探し | 2024-10 |
| 位置づけ | Basic Pitch / MuseScore と並列の「下書き」 | 2025–26 |

**長期ヘビーユーザーの熱狂レビューはX上ほぼ見つからなかった。**

### 3.2 AudioScore
- 「AudioScore is **$200**. MuseScore is free… No brainer」（2024-05）
- Sibelius＋PhotoScore＋AudioScore バンドル販促は継続（再販・セール）
- 単独の「買い切りで大満足」長期報告は希少。**記譜スイート付属のレガシー資産**寄り

### 3.3 スコアメーカー（KAWAI・日本）— 相対的に実ユーザーが厚い

| 声 | 内容 | 示唆 |
|----|------|------|
| 2026-06 | **10Pro所持**、PDF読込精度に驚き。ZEROフリーと併用 | 長期保有 |
| 2026-07 | ZERO再導入で精度向上、MusicXML→GuitarPro | 継続利用 |
| 2026-07 | スキャン解像度要求が厳しく業務複合機でも警告 | UX不満 |
| 2026-07 | AIでMIDI生成→**スコアメーカーで手直し** | **AI時代＝修正レイヤ** |
| 2026-06 | 自動認識＋音符密集ミス→手直し。専門外で学生に頼みたい | 限界を知りつつ使用 |

※クーポン／アフィリエイトは多数あるが上記は利用実態として採用。

### 3.4 仮説判定

| 仮説 | 判定 |
|------|------|
| 買い切り市場は死んだ | **否定（特にJP）** |
| 長期利用者が熱狂的に擁護 | **弱い**（淡々とした機能報告中心） |
| AI SaaSに完全置換 | **否定**（AI下書き→買い切り修正） |
| 英語圏AnthemScoreは高満足 | **購入後トラブルの生声が目立つ** |

## 4. MuseScore / Dorico エコシステムと採譜AI

### 4.1 支配的ワークフロー（日・英・中で共通、2024–2026）

```
音源 → (Demucs等でステム) → Basic Pitch / AnthemScore → MIDI
     → MuseScore で譜面編集・PDF化
```

実例:
- 「Demucs→Basic Pitch→MuseScore CLI」（2026-02, JP）
- 「Sunoステム→AnthemScore/Basic Pitch→MuseScore」（2026-06, JP）
- Basic Pitchサイト→MIDI→MuseScore（2024-08, JP）
- 中国語でも同系統の推奨が流通

### 4.2 プラグイン開発

| 発見 | 結果 |
|------|------|
| MuseScoreプラグインが**採譜AIを直接呼ぶ** | **見つからなかった** |
| 記譜補助プラグイン | あり（中心軸色分け等、2026-06） |
| QML開発ハードル | 「documentどこ…AI丸投げでも基礎必要」 |
| Dorico | Cubase→MusicXML/MIDIの自作連携（2026-07）。**採譜AIではない** |
| Sibelius | AudioScore/PhotoScoreは隣接バンドルとして残存 |

### 4.3 仮説判定

| 仮説 | 判定 |
|------|------|
| 記譜ソフト内にAI採譜が深く統合 | **否定**（コミュニティ語彙上） |
| MIDI/MusicXMLをハブに自前で組む | **強く裏付け** |
| MuseScore無料が有料AudioScoreの代替 | **裏付け** |
| DoricoユーザーがAI採譜ネイティブを積極利用 | **未確認** |

**プロダクト示唆:** 「MuseScore用プラグイン」より **編集可能な MusicXML/MIDI を確実に吐き、既存記譜ソフトに吸わせる** 方が既存行動と一致。
