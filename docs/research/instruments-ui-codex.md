# 音楽計器UIパターン調査まとめ

## 1. 針式 vs デジタル数値表示

**確立知見:**  
数値を正確に読むだけならデジタルが強い。一方、変化方向・変化率・「目標へ寄せる」操作では、連続的な空間位置を持つアナログ/グラフィック表示が有利になりやすい。

- 車載スピードメーター研究では、デジタル/冗長表示は「現在値の報告」「目標値との比較」に強く、アナログ/冗長表示は「動的な速度変化の検出」に強いと整理されている。DOI: `10.1016/j.apergo.2017.05.013` ([sciencedirect.com](https://www.sciencedirect.com/science/article/pii/S0003687017301242))
- 同研究の本文でも、デジタルは絶対値読み取り、アナログは変化検出で効率的というタスク依存性が明示されている。([sciencedirect.com](https://www.sciencedirect.com/science/article/pii/S0003687017301242))
- MIL-STD-1472系の人間工学基準でも、変化パターンが重要な場合は数値だけにせず、代表的/グラフィック表示を使うべきとされる。
- ただし航空計器では「アナログが常に優位」ではない。操縦席の客室与圧表示比較では、統合されたデジタル表示が状況認識で好まれた研究もある。DOI: `10.1002/hfm.20826` 

**採譜アプリへの示唆:**  
ピッチ偏差のような「中央ゼロへ寄せる」タスクは針式/中央ゼロメーターが合う。ただし、採譜結果の編集画面では最終的な数値や音名も必要なので、**針 + cents数値**の冗長表示が妥当。

## 2. ストロボチューナー

**確立知見:**  
ストロボ表示は、入力音と基準周波数の差を「模様の移動速度・方向」として見せ、合うと模様が静止する。つまり、誤差の大きさを数字で読むより、動きを止めるタスクとして扱える。

- Peterson公式は、ストロボ模様が左右/上下に動き、静止に近づけることで調律すると説明している。([petersontuners.com](https://www.petersontuners.com/support/))
- 同公式は、Petersonストロボチューナーの検出・表示精度を `0.1 cent` と説明している。([petersontuners.com](https://www.petersontuners.com/support/))
- ストロボチューナー特許でも、可視パターンの移動速度が周波数差を示し、目標はパターンをほぼ静止させることだと説明されている。([developer.apple.com](https://developer.apple.com/design/human-interface-guidelines?lang=en&utm_source=openai))

**採譜アプリへの示唆:**  
リアルタイム録音中のピッチ安定性確認、長音・ビブラート・ポルタメントの確認には有効。通常の採譜一覧に常時出すと過密なので、**詳細検査モード/選択ノートの解析ビュー**に限定推奨。

## 3. VUメーターとピークメーター

**確立知見:**  
VUは瞬間ピークではなく、音声/音楽プログラムの平均的な大きさを読むための標準化された表示。IEC 60268-17は電気機械式の標準音量指示器の特性・測定法を定め、ピークメーターは別規格対象としている。([standards.globalspec.com](https://standards.globalspec.com/std/787502/iec-60268-17))

- AESはVUメーターを約 `300ms` の上昇/下降を持つ平均系メーターとして説明し、1940年代の標準化経緯と、聴感上の大きさとの相関を確認した研究背景に触れている。([aes.org](https://aes.org/resources/audio-topics/loudness-project/learn-more/))
- ピークメーターは瞬間/短時間ピークを示し、クリッピング回避に重要。AESはピークメーターとVUの役割差も説明している。([aes.org](https://aes.org/resources/audio-topics/loudness-project/learn-more/))

**採譜アプリへの示唆:**  
録音レベル監視には **VU風の平均レベル + ピーク/clip LED** が強い。採譜AIでは入力が小さすぎる/割れている問題が精度に直結するため、これは採用推奨。

## 4. スキューモーフィズムと実機メタファー

**確立知見:**  
メタファーは学習転移に効くが、物理的見た目の再現そのものが価値ではない。機能との対応がズレると逆に混乱を生む。

- UIメタファー研究では、ユーザーの既有知識を使って対象ドメインのメンタルモデル形成を助ける一方、対応関係の設計が重要とされる。DOI: `10.1016/B978-0-444-70536-5.50008-7` 
- メタファー型訓練では、既有スキーマと対象インターフェースの構造的近さが学習成績に関係する。DOI: `10.1177/154193120104500608` 
- 近年の研究は、スキューモーフィズム vs フラットの二分法より、視覚的親しみやすさ・操作との物理的適合・適切な記号性が重要と結論づけている。DOI: `10.1016/j.ijhcs.2020.102435` 

**現代HIGの立場:**  
Appleは現在も「人が知っているメタファーや慣習を使う」ことを推奨するが、過度に直訳的/抽象的なメタファーや、慣習と違う挙動は避ける立場。([developer.apple.com](https://developer.apple.com/videos/play/wwdc2026/250/?time=804&utm_source=openai))  
Material Designも物理世界を抽象化した「material metaphor」を採用し、影・奥行き・動きで意味を伝えるが、実物コピーではなく一貫した空間モデルとして扱う。([m1.material.io](https://m1.material.io/material-design/elevation-shadows.html))

**音楽ソフト文脈:**  
DAWやプラグインは、初期にはミキサー/テープ/アウトボード機材の見た目で学習転移を助けたが、デジタル固有機能には新しいメタファーが必要という批評がある。([arpjournal.com](https://www.arpjournal.com/asarpwp/theme/skeuomorphism/))

## 5. グランス性

**確立知見:**  
一瞥把握には、数値密度よりも、状態・異常・方向・関係を空間的/パターン的に見せることが重要。

- Janus mini-dashboard研究は、複数システムを素早くスキャンして問題へ注意を向けるため、データ過多の表示を避けるべきとする。DOI: `10.1177/1541931213571447` ([journals.sagepub.com](https://journals.sagepub.com/doi/10.1177/1541931213571447))
- プロセス制御の概要表示研究では、機能関係に基づく定性的グラフィック表示が、数値中心の模式表示より状況認識を高めた。DOI: `10.1177/1555343412440694` ([journals.sagepub.com](https://journals.sagepub.com/doi/10.1177/1555343412440694))
- ダッシュボード研究でも、情報量が増えるほど認知負荷が上がることが確認されている。DOI: `10.1002/col.22768` 

## 採用判断

| 対象 | 判断 | 推奨パターン |
|---|---:|---|
| ピッチ偏差 | 採用推奨 | 中央ゼロ針 + cents数値。詳細ビューにストロボ。 |
| 信頼度表示 | 条件付き | 針より、低/中/高のバー・ドット・色付き信頼帯。%数値は補助。 |
| 録音レベル | 採用推奨 | VU風平均レベル + peak/clip LED。入力品質の警告に直結させる。 |
| テンポ/ビート | 条件付き | BPMは数値、位相/拍はLED階段・パルス・移動マーカー。針式BPMは非推奨。 |
| 全体UI | 条件付き | 実機風全面スキンは避け、静かな工房UIの中に計器パターンを局所的に埋め込む。 |

**結論:**  
AI採譜アプリには、計器メタファーを「装飾」ではなく「タスク適合表示」として取り込む価値がある。採用すべきは、ピッチ偏差・録音レベル・ビート位相のように、変化・中心合わせ・状態監視が重要な場所。非推奨なのは、プラグイン風の全面スキューモーフィックUIや、正確な表形式確認まで針に置き換える設計。
