# フィールド録音モード(C8) SNR劣化カーブ — 2026-07-19 実測

対象: 合成メロディ(MELODY_SIMPLE・BPM100・sine) + 雑音混入、mono(pYIN)エンジン + 降噪(スペクトラルゲート) + SNR適応選択。
指標: Note F1(音高+onset±80ms)。ベースライン封緘値はこの実測から余裕を引いた回帰ガード(test_field_mode.py)。

| 条件 | F1(降噪なし・素) | F1(フィールドモード) | 封緘フロア |
|---|---|---|---|
| クリーン | 1.000 | 1.000 | ≥0.8 |
| 白色雑音 SNR20dB | 1.000 | 1.000 | ≥0.9 |
| 白色雑音 SNR10dB | 0.118 | **1.000** | ≥0.85 |
| 白色雑音 SNR5dB | 0.000 | 0.000(検出消失) | ゴミ非出力+very_noisy報告 |
| ピンク雑音 SNR20dB | 1.000 | 1.000 | ≥0.9 |
| ピンク雑音 SNR10dB | 0.968→選択で0.316(較正前) | **1.000** | ≥0.85 |
| ピンク雑音 SNR5dB | 0.000 | **0.815** | ≥0.6 |
| 残響(t60=0.6s) | — | ≥0.6ガード | ≥0.6 |
| 打撃混入 SNR10dB | — | ≥0.5ガード | ≥0.5 |

## 発見(正直な記録)

1. **降噪(スペクトラルゲート)が決定打**: 白SNR10dBで0.118→1.000、ピンクSNR5dBで0→0.815。pYINの雑音下崩壊は前処理で大幅に救済可能
2. **選択フィルタの初期設計は絞めすぎだった**: mono(pYIN)は雑音下でも幽霊をほぼ出さないため、確信度要求の過度な引き上げは良品を殺すだけ(ピンクSNR10で0.968→0.316の実測)。較正して軽い絞り+極短除去に変更
3. **白色SNR5dBはpYINの実測限界**: 検出が消失する。この領域の要求は「間違った音符の洪水を出さない」に定義(検出数≤正解数・very_noisy報告)— 「拾えないものは拾えないと正直に言う」の実装
4. **SNR推定器は内部プロキシ**: クリーン疎音源で≈9dBに飽和する特性を実測し、バケット(clean≥8/noisy≥6)をそのスケールに較正。絶対SNRではないことをコードに明記
5. 限界: poly(basic-pitch)エンジンでの劣化カーブは未測定(次の課題)。実フィールド録音(非合成)はG1のU6セルで検証

# フィールド録音モード(C8/F-108) 分類タグ+選択抽出率/誤音符化率 — 2026-07-20 実測(Issue #60)

要件v2.7 F-108の受入条件をテスト化し、選択抽出率・誤音符化率のベースラインを封緘なしで記録した(初回=記録のみ)。

## 分類タグ(6種)のテスト化

`stem/field.py::classify_segment` が音事件を6タグ(pitched_stable / pitched_transient / noisy / speech / poly / inharmonic)に分類する分類器を新設。各タグの代表合成音で固定(`tests/test_field_acceptance.py`)。noisy/inharmonic/speech は `is_notable=False`(音符化せず音響オブジェクトとして保持)、pitched 2種+poly は `is_notable=True`。既定は単音抽出優先で poly は `gate_by_class` が音符化を保留(allow_poly=Trueで解禁)。

分類器の正直な限界: 褐色雑音は低域集中で平坦度≈0となり、乱数の引き次第で稀に pitched へ化ける(実測 27/30 seed が noisy)。合成上の病理であり、実フィールドの低域ランブル(交通/空調)は純褐色より構造を持つため本番リスクとは別物。ピンクは30/30・白色は全seedで安定。

## 選択抽出率・誤音符化率(bench_field_extract.py)

- **選択抽出率** = 正解メロディ線ノートの recall(音高一致+onset±100ms)。
- **誤音符化率** = 出力ノートのうち正解に対応しない割合(=1-precision)。
- **雑音のみ誤発火** = 正解ゼロの純雑音を採譜したときの余剰ノート数÷正解ノート数。
- 正解は伴奏つきPD曲の単声スカイライン(最高音メロディ線)を抽出し、その**メロディ線だけを単音合成**して雑音を混ぜた(多声そのままの合成はpretty_midi+pYINが低音に張り付き、クリーンでも抽出≒0となり雑音耐性を測れないため。単音経路の評価に条件を揃えた)。
- 対象3曲(民謡・単声メロディが明瞭): furusato / sakura / hamabe_no_uta。各先頭15秒。雑音3種(白色/ピンク/打撃)×SNR 20/10/5dB。

## 発見(正直な記録)

1. **雑音のみ誤発火は全27条件で0.000**: 純雑音(白/ピンク/打撃)を採譜しても余剰ノートを1つも出さない。「拾えないものは拾えないと正直に言う」がメロディ合成条件でも成立。
2. **SNR20/10dBは選択抽出率≈0.90**: 単声化条件では白/ピンク/打撃いずれもメロディの9割を保存。誤音符化率もほぼ0。
3. **白色SNR5dBで抽出が消失(3曲とも0.000)**: 既存results(発見3)と一致するpYINの実測限界。ピンク・打撃はSNR5でも0.7〜0.93を保つ(白色が最も過酷)。
4. **SNR5dBの誤音符化率上昇(打撃0.32〜0.56)**: 低SNRの打撃残差がonsetを生み幽霊が増える。選択フィルタは軽い絞りに較正済み(既存発見2)のため、この帯は幽霊とのトレードオフが残る。閾値は封緘せず今回は記録のみ(受入条件どおり)。
5. **限界**: 正解はメロディ線に限定(多声混合からの単音抽出そのものの劣化はF-108のpolyタグ=オンデマンド分解側の課題として分離)。実フィールド録音(非合成)はG1のU6セルで検証。褐色雑音の分類は上記のとおり稀に pitched へ化ける合成上の限界あり。
