# フィールド録音モード(C8) SNR劣化カーブ — 2026-07-19 実測

対象: 合成メロディ(MELODY_SIMPLE・BPM100・sine) + 雑音混入、mono(pYIN)エンジン + 降噪(スペクトラルゲート) + SNR適応選択。
指標: Note F1(音高+onset±80ms)。ベースライン封緘値はこの実測から余裕を引いた回帰ガード(test_field_mode.py)。

| 条件 | F1(降噪なし・素) | F1(フィールドモード) | 封緘フロア |
|---|---|---|---|
| クリーン | 1.000 | 1.000 | ≥0.8 |
| 白色雑音 SNR20dB | 1.000 | 1.000 | ≥0.9 |
| 白色雑音 SNR10dB | 0.118 | **1.000** | ≥0.85 |
| 白色雑音 SNR5dB | 0.000 | 0.000(検出消失) | ゴミ非出力+very_noisy報告 |
| ピンク雑音 SNR20dB | 1.000 | 1.000 | ≥0.9 |
| ピンク雑音 SNR10dB | 0.968→選択で0.316(較正前) | **1.000** | ≥0.85 |
| ピンク雑音 SNR5dB | 0.000 | **0.815** | ≥0.6 |
| 残響(t60=0.6s) | — | ≥0.6ガード | ≥0.6 |
| 打撃混入 SNR10dB | — | ≥0.5ガード | ≥0.5 |

## 発見(正直な記録)

1. **降噪(スペクトラルゲート)が決定打**: 白SNR10dBで0.118→1.000、ピンクSNR5dBで0→0.815。pYINの雑音下崩壊は前処理で大幅に救済可能
2. **選択フィルタの初期設計は絞めすぎだった**: mono(pYIN)は雑音下でも幽霊をほぼ出さないため、確信度要求の過度な引き上げは良品を殺すだけ(ピンクSNR10で0.968→0.316の実測)。較正して軽い絞り+極短除去に変更
3. **白色SNR5dBはpYINの実測限界**: 検出が消失する。この領域の要求は「間違った音符の洪水を出さない」に定義(検出数≤正解数・very_noisy報告)— 「拾えないものは拾えないと正直に言う」の実装
4. **SNR推定器は内部プロキシ**: クリーン疎音源で≈9dBに飽和する特性を実測し、バケット(clean≥8/noisy≥6)をそのスケールに較正。絶対SNRではないことをコードに明記
5. 限界: poly(basic-pitch)エンジンでの劣化カーブは未測定(次の課題)。実フィールド録音(非合成)はG1のU6セルで検証
